## Directory Purposes

**`banking_api_server/config/`:**
- Purpose: Static and runtime-configurable OAuth parameters
- Key files: `oauth.js` (admin OAuth client config), `oauthUser.js` (user OAuth client config), `scopes.js` (scope catalog and role maps), `runtimeSettings.js`, `hosting.js`

**`banking_api_server/data/`:**
- Purpose: In-memory banking data; loaded at startup; never persists to disk
- Key files: `store.js` (singleton dataStore), `bootstrapData.json` (seed data), `sampleData.js`, `persistent/` (demo snapshot support)

**`banking_api_server/middleware/`:**
- Purpose: Reusable Express middleware inserted in the global pipeline
- Key files: `auth.js` (`authenticateToken` — validates JWT or session token, sets `req.user`), `scopeEnforcement.js` (`requireScopes([...])`), `tokenRefresh.js` (`refreshIfExpiring`), `delegationAuditLogger.js`, `correlationId.js`, `activityLogger.js`, `actClaimValidator.js`

**`banking_api_server/routes/`:**
- Purpose: Express Router modules, one per domain area
- Key files: `oauth.js` (admin PKCE login/callback), `oauthUser.js` (user PKCE login/callback), `accounts.js`, `transactions.js`, `users.js`, `admin.js`, `ciba.js`, `authorize.js`, `tokens.js`, `featureFlags.js`, `mcpInspector.js`, `bankingAgentNl.js`, `agentIdentity.js`
- Note: `server.js` mounts routes with `app.use('/api/<path>', router)` — see route mount list below

**`banking_api_server/services/`:**
- Purpose: All business logic, external calls, token management
- Key files: `configStore.js` (runtime config singleton), `oauthService.js` (admin token ops), `oauthUserService.js` (user token ops), `agentMcpTokenService.js` (RFC 8693 token chain), `mcpWebSocketClient.js` (WebSocket JSON-RPC client to MCP server), `tokenValidationService.js` (JWKS validation), `upstashSessionStore.js` (Upstash Redis session store), `authStateCookie.js` (signed `_auth` cookie for cold-start recovery), `bffSessionGating.js`, `pingOneAuthorizeService.js`, `mcpToolAuthorizationService.js`, `exchangeAuditStore.js`

**`banking_api_ui/src/components/`:**
- Purpose: Flat collection of React components — one file per UI surface (74 total)
- Key files: `Dashboard.js`, `UserDashboard.js`, `BankingAgent.js`, `Config.js`, `Accounts.js`, `Transactions.js`, `Users.js`, `DelegatedAccessPage.js`, `CIBAPanel.js`, `McpInspector.js`, `EmbeddedAgentDock.js`, `EducationBar.js`
- Convention: Components are NOT organized into sub-folders (except `shared/` and `education/`)

**`banking_api_ui/src/services/`:**
- Purpose: API layer and UI-side in-memory stores
- Key files: `bffAxios.js` (credential-bearing Axios for BFF), `apiClient.js` (generic API helper), `bankingAgentService.js` (MCP tool calls), `mcpFlowSseClient.js` (SSE for live MCP flow events), `apiTrafficStore.js`, `mcpCallStore.js`, `toastLogStore.js`

**`banking_mcp_server/src/tools/`:**
- Purpose: MCP tool registration and execution
- Key files: `BankingToolRegistry.ts` (static tool catalog), `BankingToolProvider.ts` (execution engine), `BankingToolValidator.ts` (param validation), `AuthorizationChallengeHandler.ts` (missing scope detection → challenge response)

**`banking_mcp_server/src/server/`:**
- Purpose: WebSocket server and MCP protocol handling
- Key files: `BankingMCPServer.ts` (WebSocket listen/accept), `MCPMessageHandler.ts` (JSON-RPC dispatch: `initialize`, `tools/list`, `tools/call`), `HttpMCPTransport.ts`

## Key File Locations

**Entry Points:**
- `banking_api_server/server.js`: BFF Express app (local dev and Vercel serverless via `api/handler.js`)
- `banking_api_ui/src/index.js`: React SPA bootstrap
- `banking_api_ui/src/App.js`: Root router with all route-to-component mappings
- `banking_mcp_server/src/index.ts`: MCP server startup
- `api/handler.js`: Vercel serverless adapter

**Configuration:**
- `vercel.json`: Deployment routes, build commands, security headers
- `banking_api_server/services/configStore.js`: Runtime OAuth config (read via `configStore.getEffective(key)`)
- `banking_api_server/config/oauth.js`: Admin OAuth client static config
- `banking_api_server/config/oauthUser.js`: User OAuth client static config
- `banking_api_server/config/scopes.js`: Scope catalog, route-scope map, role-to-scope map
- `banking_api_ui/.env`: UI env vars including `REACT_APP_API_PORT`

**Core Business Logic:**
- `banking_api_server/data/store.js`: In-memory data store singleton
- `banking_api_server/middleware/auth.js`: `authenticateToken` + `requireScopes`
- `banking_api_server/services/agentMcpTokenService.js`: RFC 8693 token chain resolution
- `banking_api_server/services/mcpWebSocketClient.js`: BFF → MCP WebSocket client, `MCP_TOOL_SCOPES` map
- `banking_mcp_server/src/tools/BankingToolRegistry.ts`: MCP tool definitions and scope requirements
- `banking_mcp_server/src/tools/BankingToolProvider.ts`: MCP tool execution

**Testing:**
- `banking_api_ui/src/__tests__/`: UI unit tests
- `banking_api_ui/src/context/__tests__/`: Context unit tests
- `banking_api_ui/src/services/__tests__/`: Service unit tests
- `banking_mcp_server/jest.config.js`: MCP server Jest config

## Naming Conventions

**Files (BFF — CommonJS):**
- Routes: `camelCase.js` matching the domain noun (`accounts.js`, `oauthUser.js`)
- Services: `camelCase.js` with pattern suffix (`oauthService.js`, `configStore.js`, `upstashSessionStore.js`)
- Middleware: `camelCase.js` (`auth.js`, `tokenRefresh.js`, `correlationId.js`)

**Files (MCP — TypeScript):**
- Classes: `PascalCase.ts` matching the class name (`BankingToolRegistry.ts`, `BankingSessionManager.ts`)
- Interfaces: `camelCase.ts` by domain (`auth.ts`, `mcp.ts`) in `interfaces/` and `types/`
- Barrel files: `index.ts` in every `src/` subdirectory

**Files (React SPA):**
- Components: `PascalCase.js` matching the default export (`Dashboard.js`, `CIBAPanel.js`)
- Services: `camelCase.js` with pattern suffix (`bffAxios.js`, `bankingAgentService.js`, `apiTrafficStore.js`)
- CSS: Named after component (`BankingAgent.css`, `CIBAPanel.css`)
- Contexts: `PascalCaseContext.js` (`SpinnerContext.js`, `TokenChainContext.js`)

**Directories:**
- BFF subdirs: lowercase (`routes/`, `services/`, `middleware/`, `config/`, `data/`)
- MCP subdirs: lowercase (`tools/`, `auth/`, `server/`, `storage/`, `utils/`, `interfaces/`, `types/`)
- UI subdirs: lowercase (`components/`, `services/`, `hooks/`, `context/`, `config/`, `utils/`)

## Where to Add New Code

**New BFF API route:**
1. Create `banking_api_server/routes/<noun>.js` (Express Router)
2. `require` it in `banking_api_server/server.js` and add `app.use('/api/<noun>', ...)` — after the protected route block if requires `authenticateToken`
3. Add required scopes using `requireScopes([...])` from `middleware/scopeEnforcement.js`

**New service / business logic:**
- Add to `banking_api_server/services/<serviceName>.js`
- Read config via `configStore.getEffective(key)` — never directly from `process.env`
- Import the service in the relevant route file (singleton pattern: `require('./services/...')`)

**New MCP tool:**
1. Add entry to `BankingToolRegistry.TOOLS` in `banking_mcp_server/src/tools/BankingToolRegistry.ts` with `name`, `description`, `inputSchema`, `requiredScopes`, `handler`
2. Add a corresponding `execute<ToolName>()` method to `BankingToolProvider.ts`
3. Add scope entries to `MCP_TOOL_SCOPES` in `banking_api_server/services/mcpWebSocketClient.js`

**New React component:**
- Add `banking_api_ui/src/components/<ComponentName>.js` (and optional `<ComponentName>.css`)
- Register route in `banking_api_ui/src/App.js` `<Routes>` block if it's a page-level component

**New React context:**
- Add `banking_api_ui/src/context/<Name>Context.js`
- Wrap with provider in `App.js`

**Shared utilities:**
- BFF: `banking_api_server/utils/`
- React: `banking_api_ui/src/utils/`
- MCP: `banking_mcp_server/src/utils/`
