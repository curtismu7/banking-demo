Add a structured passthrough so the UI can detect the step-up requirement and trigger CIBA.

Output: BankingToolProvider.ts catches step_up_required and returns `{ step_up_required: true, step_up_method, message }`. TypeScript builds clean.

In BankingToolProvider.ts, find the THREE catch blocks in `executeSpecificTool()` that handle `consent_challenge_required` (they're in the deposit, withdraw, and transfer cases at roughly lines 373, 422, and 478).

**In each catch block, BEFORE the existing `consent_challenge_required` check, add:**

```typescript
if (error instanceof BankingAPIError && error.errorCode === 'step_up_required') {
  const axiosData = (error as any).axiosError?.response?.data || {};
  const stepUpMethod: string = typeof axiosData.step_up_method === 'string'
    ? axiosData.step_up_method
    : 'ciba';
  const payload = {
    error: 'step_up_required',
    step_up_required: true,
    step_up_method: stepUpMethod,
    message: `This transaction requires additional authentication (${stepUpMethod.toUpperCase()}). Please complete the step-up verification to proceed.`,
    amount_threshold: typeof axiosData.amount_threshold === 'number' ? axiosData.amount_threshold : null,
  };
  return {
    content: [{ type: 'text' as const, text: JSON.stringify(payload) }],
    step_up_required: true,
    step_up_method: stepUpMethod,
    isError: false,
  };
}
```

**Do not** change or reorder the `consent_challenge_required` check that follows.
**Do not** change `createAuthChallengeResult` or the authChallenge path.
**Do not** change the function signatures or any other catch blocks.
