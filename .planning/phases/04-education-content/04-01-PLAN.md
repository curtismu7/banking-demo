Three tabs:

**Tab 1 — id: "what", label: "What changed"**
Content covers 5 main OIDC 2.1 changes vs OIDC Core 1.0:
1. PKCE mandatory for all Authorization Code flows (was optional in 1.0)
2. Implicit flow removed (id_token/token response_type deprecated)
3. Resource Indicators (RFC 8707) alignment — audience binding is now expected
4. Refresh token rotation recommended by default
5. Nonce requirement tightened — AS must reject replayed nonces

Include a comparison table:
| Feature | OIDC Core 1.0 | OIDC 2.1 |
|---------|--------------|----------|
| PKCE | Optional | Required |
| Implicit flow | Allowed | Removed |
| Resource indicators | Optional | Recommended |
| Refresh rotation | Optional | Recommended |
| Nonce replay protection | Recommended | Required |

**Tab 2 — id: "agents", label: "Why AI agents care"**
Explain why each OIDC 2.1 change specifically matters for AI agent architectures:
- Mandatory PKCE prevents code interception by rogue agents on shared systems
- No implicit flow means agents cannot receive tokens in URL fragments (which are logged)
- Resource indicators enable audience-scoped tokens for each upstream resource (like MCP servers)
- Refresh rotation means compromised agents cannot silently refresh indefinitely
- Include a callout: "This demo is OIDC 2.1-aligned: BFF holds all tokens, browser never receives a token, PKCE always used"
