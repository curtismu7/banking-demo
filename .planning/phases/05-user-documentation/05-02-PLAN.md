**BX-Finance-CIBA-Flow.drawio**

This diagram shows the CIBA (Client-Initiated Backchannel Authentication) flow used for out-of-band agent approval.

**Actors (left to right):**
1. AI Agent (BankingAgent.js)
2. BFF (banking_api_server)
3. PingOne
4. User Device (email / push notification)

**Steps to show:**
1. AI Agent → BFF: "Initiate high-value operation (e.g. transfer $1,000)"
2. BFF → PingOne: "POST /bc-authorize: login_hint=user@email, scope=banking:write, binding_message='Approve transfer'"
3. PingOne → BFF: "auth_req_id, expires_in=120" (dashed return)
4. PingOne → User Device: "Push notification / email: 'Approve transfer?'"
5. BFF → AI Agent: "{ cibaRequired: true, auth_req_id }" (dashed return)
6. AI Agent → BFF: "Poll: GET /api/ciba/status?auth_req_id=... (every 5s)"
7. User Device → PingOne: "User clicks Approve"
8. PingOne → BFF: "access_token (elevated)" on next poll (dashed return arrow back to polling)
9. BFF → AI Agent: "{ approved: true, token: '...' }" (dashed return)
10. AI Agent → BFF: "Retry original tool call with elevated token"
11. BFF → AI Agent: "Tool result returned"

**Annotations:**
- Yellow note on step 2: "RFC 9126 / CIBA: backchannel — no browser redirect needed"
- Yellow note on step 6: "Polling interval: 5 seconds; expiry: 120 seconds (configurable)"



**BX-Finance-TokenExchange-Flow.drawio**

This diagram shows RFC 8693 token exchange — two paths on the same diagram (1-exchange on the left half, 2-exchange on the right half with a divider).

Use a page title "RFC 8693 Token Exchange — 1-Exchange vs 2-Exchange".

**1-EXCHANGE PATH (left side):**

Actors: BFF, PingOne Token Endpoint

Steps:
1. BFF: "Has user access_token (from session)" (self-activation box)
2. BFF → PingOne: "POST /token: grant=urn:ietf:params:oauth:grant-type:token-exchange, subject_token=user_at, requested_token_type=access_token, audience=banking_mcp_01"
3. PingOne → BFF: "MCP access_token (aud=banking_mcp_01, scope narrowed)" (dashed return)
4. BFF → MCP Server: "Tool call with Bearer MCP_token"

**2-EXCHANGE PATH (right side):**

Actors: BFF, Agent (Client Credentials), PingOne Token Endpoint A, PingOne Token Endpoint B, MCP Server

Steps:
1. BFF → PingOne_A: "POST /token: grant=client_credentials, client_id=agent_client (actor token)"
2. PingOne_A → BFF: "agent_actor_token (sub=agent@client)" (dashed return)
3. BFF → PingOne_B: "POST /token: grant=token-exchange, subject_token=user_at, actor_token=agent_actor_token, audience=banking_mcp_01"
4. PingOne_B → BFF: "MCP access_token with act claim: { sub: user, act: { sub: agent } }" (dashed return)
5. BFF → MCP Server: "Tool call with Bearer MCP_token (act claim present)"

**Annotations:**
- Yellow note box on 2-exchange result: "RFC 8693 §4.1: act claim proves delegation chain — user authorized agent to act on their behalf"
- Yellow note box on 1-exchange: "simpler path — use when agent identity doesn't need to be in the token"
- Blue box divider between left and right paths labeled "Feature flag: USE_AGENT_ACTOR_FOR_MCP / ff_two_exchange_delegation"

Use same style conventions as existing docs/BX-Finance-1-Exchange-Delegated-Chain.drawio.
