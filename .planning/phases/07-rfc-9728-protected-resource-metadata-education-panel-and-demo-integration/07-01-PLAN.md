  <action>
Create a new Express router file at `banking_api_server/routes/protectedResourceMetadata.js`.

It exports a single router with two GET handlers:

1. `GET /` — the root handler used when mounted at `/.well-known/oauth-protected-resource`
2. `GET /metadata` — the proxy handler used when mounted at `/api/rfc9728`

Both handlers call the same `buildMetadata(req)` helper function which returns the RFC 9728 JSON document. The `resource` field is built from `PUBLIC_APP_URL` env var (if set) or falls back to constructing it from `req.protocol + '://' + req.get('host') + '/api'`.

The `buildMetadata(req)` function:
```js
function buildMetadata(req) {
  const baseUrl = process.env.PUBLIC_APP_URL ||
    `${req.protocol}://${req.get('host')}`;
  const envId = process.env.PINGONE_ENVIRONMENT_ID || '';
  const region = process.env.PINGONE_REGION || 'com';
  const asList = envId
    ? [`https://auth.pingone.${region}/${envId}/as`]
    : [];

  return {
    resource: `${baseUrl}/api`,
    ...(asList.length > 0 ? { authorization_servers: asList } : {}),
    bearer_methods_supported: ['header'],
    scopes_supported: [
      'banking:read',
      'banking:write',
      'banking:admin',
      'banking:accounts:read',
      'banking:transactions:read',
      'banking:transactions:write',
    ],
    resource_name: 'BX Finance Banking API',
    resource_documentation: 'https://datatracker.ietf.org/doc/html/rfc9728',
  };
}
```

Both route handlers:
- Set `Content-Type: application/json`
- Return `res.json(buildMetadata(req))`
- No authentication required (public endpoints per RFC 9728 §3)
- No session or token reading

Module exports: `module.exports = router;`

Wait — it's cleaner to just extract the handler. Actually the simplest approach:

```js
const protectedResourceMetadataRoutes = require('./routes/protectedResourceMetadata');
```

Then at the well-known registration block:
```js
// RFC 9728 — Protected Resource Metadata (public, no authentication)
app.use('/.well-known/oauth-protected-resource', protectedResourceMetadataRoutes);
app.use('/api/rfc9728', protectedResourceMetadataRoutes);
```

This works because the route handler is registered at `GET /` in the router — when mounted at `/.well-known/oauth-protected-resource`, a GET to that exact path matches `GET /` in the subrouter.
