<objective>
Create a visual token exchange flow diagram component that shows the RFC 8693 token exchange process for both 1-exchange (subject-only) and 2-exchange (subject + agent actor) paths.

Purpose: Educate users on the difference between the two exchange modes by showing a clear, labeled visual flow of how tokens move through the system (Browser → BFF → PingOne → MCP Server).

Output: 
- `TokenExchangeFlowDiagram.jsx` component that renders SVG-based flow diagrams
- `TokenExchangeFlowDiagram.css` with BEM-based styling
- Component accepts `mode` prop ('single' or 'double') and renders appropriate flow
- Color-coded token types and RFC-labeled steps
</objective>

  <action>
Create a React component (TokenExchangeFlowDiagram.jsx) that renders an SVG-based flow diagram showing the 1-exchange token flow path.

**Structure:**
- Component accepts `mode` prop ('single' or 'double'), defaults to 'single'
- For 'single' mode, render the 1-exchange flow (this task)
- Render actor/participant boxes: User, BFF, PingOne, MCP Server
- Show token flow as labeled arrows between boxes
- Label each step with RFC reference (e.g., "RFC 8693 §2.1: Subject Token Exchange")
- Use semantic class names for CSS targeting (.tefd-box, .tefd-token, .tefd-arrow, .tefd-label)

**1-Exchange flow path:**
1. User (Browser) — "User logs in"
2. BFF receives: "User Token (subject=user_id)" with may_act claim
3. BFF calls PingOne /token — "Request: subject_token (narrowed to MCP aud)"
4. PingOne returns: "MCP Token (sub=user_id, no act claim)"
5. MCP Server receives: "MCP Token (subject-only, least privilege)"

**Visual design:**
- Use inline SVG (not img) for resolution independence
- Boxes: rounded rectangles with actor names (User, BFF, PingOne OAuth, MCP Server)
- Arrows: labeled with step number + description + RFC reference
- Colors: Gray for boxes, blue for token type labels
- Font: System font, 11-14px for labels
- Sizing: Render at 100% viewport width, adapt to container (no hard px widths)

**Implementation detail:**
- Export as default: `export default function TokenExchangeFlowDiagram({ mode = 'single', className = '' })`
- If mode !== 'single', return null or placeholder (2-exchange will be added in next task)
- Use CSS class `.tefd-root` as main container
- Include basic error handling: if mode is invalid, log warning and default to 'single'

**No styling in this task** — all CSS goes to Task 2 (TokenExchangeFlowDiagram.css)
