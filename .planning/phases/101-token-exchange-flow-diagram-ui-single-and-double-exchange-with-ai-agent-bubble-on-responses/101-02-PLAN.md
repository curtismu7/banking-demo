  <files>banking_api_ui/src/context/ExchangeModeContext.js</files>
  <action>
Create a React Context and Provider for managing exchange mode state globally, so TokenExchangeFlowDiagram and other components can subscribe to mode changes without prop drilling.

**Context structure:**
```javascript
const ExchangeModeContext = React.createContext({
  mode: 'single', // 'single' or 'double'
  setMode: (mode) => {},
  loading: false,
  error: null,
});

export function useExchangeMode() {
  const ctx = useContext(ExchangeModeContext);
  if (!ctx) throw new Error('useExchangeMode must be used within ExchangeModeProvider');
  return ctx;
}

export function ExchangeModeProvider({ children }) {
  const [mode, setMode] = useState('single');
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);
  
  // Load initial mode on mount
  useEffect(() => {
    axios.get('/api/mcp/exchange-mode')
      .then(r => setMode(r.data.mode === 'double' ? 'double' : 'single'))
      .catch(err => setError(err.message));
  }, []);
  
  return (
    <ExchangeModeContext.Provider value={{ mode, setMode, loading, error }}>
      {children}
    </ExchangeModeContext.Provider>
  );
}
```

**Integration points:**
- ExchangeModeToggle.js: Use `useExchangeMode()` to read/write mode instead of local state
- TokenExchangeFlowDiagram: Use `useExchangeMode()` to pass mode prop and react to changes
- AgentFlowDiagramPanel: Wrap child components with ExchangeModeProvider or use at appropriate level

**Location:** `banking_api_ui/src/context/ExchangeModeContext.js`

**No styling required** — this is pure state management.

**Implementation note:** Only create this file if one doesn't already exist. If ExchangeModeToggle already uses a context pattern, follow that instead (don't create duplicate state).

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-101-04 | Input Validation | exchange mode polling | mitigate | Validate API response mode against ['single', 'double']; default to 'single' if invalid |
| T-101-05 | XSS (education link) | onClick handlers | mitigate | Use dispatchEvent or sealed hook (not innerHTML); don't construct URLs from user input |
| T-101-06 | Accessible interactivity | Tooltip system | mitigate | Ensure tooltip text also available via aria-label; keyboard navigation for buttons |

</threat_model>
