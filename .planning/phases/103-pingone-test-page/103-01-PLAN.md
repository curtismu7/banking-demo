# Phase 103: PingOne Test Page

## Description
Build a comprehensive test page that tests all aspects of PingOne integration including APIs, Token exchange, configuration (Apps, Scopes, Resources, User attributes), and specifications. The page will have a Chase.com-style UI with fix buttons for each test. Worker token will be retrieved on startup if credentials are saved in .env.

## Dependencies
- Phase 2: token-exchange (token exchange flows must be working)
- Phase 85: chase-dashboard-styling (Chase.com design language established)

## Deliverables
- Comprehensive PingOne test page with Chase.com-style UI
- Worker token retrieval on startup with credential caching
- Tests for PingOne APIs (Apps, Scopes, Resources, Users)
- Tests for token exchange (1-exchange and 2-exchange paths)
- Fix buttons for each test to auto-resolve configuration issues
- Configuration validation for all PingOne settings
- Real-time status indicators for all PingOne components

## Estimated Duration
3-5 days

## Success Criteria
1. Test page loads and automatically retrieves worker token on startup if creds are saved
2. All PingOne API tests run successfully with clear pass/fail indicators
3. Token exchange tests (1-exchange and 2-exchange) validate end-to-end flows
4. Fix buttons successfully resolve common configuration issues
5. UI matches Chase.com design language and is responsive
6. All tests can be run independently or in batch mode
7. Test results are persisted and can be exported

## Tasks

### 103-01: Design Test Page Architecture
- Define test categories (APIs, Token Exchange, Configuration, User Attributes)
- Design component structure for test cards
- Plan worker token retrieval and caching mechanism
- Design fix button logic and API calls

### 103-02: Implement Worker Token Retrieval
- Add endpoint to get worker token from PingOne
- Implement credential caching in session/local storage
- Add startup token retrieval with fallback
- Handle token refresh automatically

### 103-03: Build PingOne API Tests
- Apps configuration test
- Scopes validation test
- Resources configuration test
- User attributes test
- Environment settings test

### 103-04: Build Token Exchange Tests
- 1-exchange path test (user token → MCP token)
- 2-exchange path test (user + agent tokens → MCP token)
- Actor token validation test
- Audience claim validation test

### 103-05: Implement Fix Buttons
- Auto-fix common configuration issues
- API calls to PingOne Management API
- Validation after fix application
- Rollback capability for fixes

### 103-06: Chase.com-Style UI
- Implement Chase.com design system
- Responsive layout for all screen sizes
- Status indicators with color coding
- Test cards with expand/collapse
- Progress indicators for batch tests

### 103-07: Integration and Testing
- End-to-end testing of all test scenarios
- Fix button validation
- Worker token retrieval testing
- UI/UX testing
- Performance testing
