  <name>Marketing Landing Page Verification</name>
  
  <what-built>
  - LandingPage.js component with header, hero section, 4 feature cards, and embedded agent dock
  - LandingPage.css with responsive styling (desktop 1024px, tablet 768px, mobile 480px)
  - Integration with EmbeddedAgentDock for marketing surface
  - Light grey background (#F5F5F5), red accents (#b91c1c), professional typography
  </what-built>

  <how-to-verify>
  **Step 1: Run the build** (automated above)
  - Verify `npm run build` exits with code 0
  - Check bundle sizes are reasonable (< 400KB JS, < 100KB CSS)

  **Step 2: Visual inspection on desktop (1024px+)**
  - Open the built landing page or visit the dev server
  - Verify header: BX Finance logo visible, nav links present, Sign In buttons clickable
  - Verify hero section: Headline "Secured AI Banking" prominent, subheadline visible, 2 CTA buttons (red + white outline)
  - Verify feature grid: 4 cards in single row (Auth Flows, RFC 8693, MCP, Agent with correct icons and descriptions)
  - Verify background: Light grey (#F5F5F5) on entire page
  - Verify agent dock: Appears bottom-right, doesn't cover content, is interactive

  **Step 3: Responsive testing - Tablet (768px)**
  - Shrink browser to ~768px width
  - Verify header adjusts, nav may hide, header actions still clickable
  - Verify hero text remains readable
  - Verify feature grid: 2 columns (not 4)
  - Verify agent dock: Positioned correctly (static or adjusted)

  **Step 4: Responsive testing - Mobile (360px)**
  - Shrink browser to ~360px width
  - Verify header stacks vertically if needed
  - Verify hero text readable (font-size scales down)
  - Verify feature grid: 1 column
  - Verify CTA buttons full-width or responsive
  - Verify agent dock: Full-width at bottom, doesn't break layout

  **Step 5: Accessibility checks**
  - Open DevTools → Accessibility tab
  - Verify all interactive elements (buttons, links) are keyboard-accessible (Tab focus visible)
  - Verify color contrast: hero headline on background passes WCAG AA (4.5:1 for text)
  - Verify image alt text / ARIA labels on feature icons
  - Verify semantic HTML: `<header>`, `<section>`, correct heading hierarchy

  **Step 6: Console health**
  - Open DevTools → Console tab
  - Verify NO red error messages
  - Acceptable: yellow warnings (e.g., unused vars) and deprecation notices
  - Unacceptable: "Cannot read property", "Uncaught", "undefined reference"

  **Step 7: Functional verification**
  - Click "Try as Admin" → navigates to /admin/login
  - Click "Try as Customer" → navigates to /login
  - Click header "Sign In" buttons → same navigation
  - Agent dock: type message → response appears (basic interaction)

  **Step 8: Dark mode**
  - Enable dark mode (System Preferences → Appearance → Dark, or browser DevTools → Rendering → emulate CSS media feature prefers-color-scheme)
  - Verify page colors adapt: background darkens, text becomes light
  - Verify all text remains readable
  - Verify red accent is still visible

  **Step 9: Cross-browser (if available)**
  - Test Chrome/Edge: Verify all styles render correctly
  - Test Firefox: Verify layout not broken
  - Test Safari: Verify responsive breakpoints work
  </how-to-verify>
