**Logic:**

   - 504 status codes
   - Network timeouts (e.g., no response in 5s)
   - ECONNREFUSED or similar network errors

2. On first 504:
   - Set modal visible
   - Start retry loop with exponential backoff: 1s, 2s, 4s, max 30s between retries

Build the modal UI component with animations and retry button.

**Component structure:**
```jsx
export default function ServerRestartModal() {
  const { isVisible, attemptCount, retryNow, closeModal } = useRestartModal();
  
  if (!isVisible) return null;
  
  return (
    <div className="modal-overlay server-restart-modal" role="dialog" aria-modal="true">
      <div className="modal-content server-restart-content">
        <div className="modal-header">
          <h2>Server is restarting</h2>
        </div>
        <div className="modal-body">
          <div className="restart-spinner"></div>
          <p>The server is temporarily unavailable. Reconnecting...</p>
          <p className="attempt-counter">Attempt {attemptCount} of 30</p>
        </div>
        <div className="modal-footer">
          <button 
            onClick={retryNow}
            className="btn btn-blue"
          >
            Retry Now
          </button>
          <button 
            onClick={closeModal}
            className="btn btn-secondary"
          >
            Dismiss
          </button>
        </div>
      </div>
    </div>
  );
}
```

**Styling (ServerRestartModal.css):**
- Fade-in animation on .modal-overlay (0.3s ease-out)
- Spinner animation (360° rotation, 1s loop, infinite)
- Disable interaction outside modal (pointer-events: none on content behind overlay)
- Attempt counter styled in secondary color
- Buttons positioned in footer with 12px gap
- z-index: 9999 (above HITL modals at 1000, regular modals at 100)

**Accessibility:**

- aria-label on spinner (hidden but announced: "Loading")
