# Phase 109 Plan 01: Fix Agent FAB Visual Jump on Placement Buttons

## Objective

Placement buttons on the Demo Data page were calling `setAgentUi(next)` which immediately updated React context and moved the FAB/dock on screen *before* the page reloaded. User sees the agent physically jump to the new position then the page reloads and renders it again cleanly.

**Fix:** In `applyAndReload()`, for `reload: true` paths, write directly to `localStorage` instead of calling `setAgentUi()`. The 350ms-delayed page reload re-inits `AgentUiModeContext` from localStorage. For `reload: false` (middle split-view), keep `setAgentUi()` — that path intentionally updates the layout live without a reload.

## Root Cause

```js
// AgentUiModeContext.js — setAgentUi() has two side effects:
// 1. localStorage.setItem('banking_agent_ui_v2', ...)
// 2. dispatchEvent(new CustomEvent('banking-agent-ui-mode'))
// → All consumers (App.js FAB, EmbeddedAgentDock) immediately re-render
```

`applyAndReload()` called `setAgentUi(next)` unconditionally before the reload, triggering both side effects and causing the visible FAB jump.

## Tasks

### Task 1: Fix applyAndReload in AgentUiModeToggle.js

**File:** `banking_api_ui/src/components/AgentUiModeToggle.js`

Replace:
```js
setAgentUi(next);
const saved = await persistBankingAgentUi(next);
```

With:
```js
if (opts.reload) {
  // Write to localStorage only; reload re-inits context from storage cleanly
  try { localStorage.setItem('banking_agent_ui_v2', JSON.stringify(next)); } catch (_) {}
} else {
  setAgentUi(next);  // keep for middle/split-view (no reload, intentional live update)
}
const saved = await persistBankingAgentUi(next);
```

**Done criteria:** FAB does not move on screen when Bottom/Left/Right/Float buttons are clicked; Middle still works live.
