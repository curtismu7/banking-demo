<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BFF → PingOne token endpoint | BFF sends MCP Exchanger client credentials to PingOne; uses correct client ID (not AI Agent ID) |
| BFF → PingOne Management API | BFF sends Worker client credentials; uses correct Worker client ID (not User/Admin ID) |
| Config storage → Runtime retrieval | Client IDs are public (no secrets); retrieved via configStore with priority chain |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-111-01 | **S**poofing | Token endpoint | mitigate | Verify PingOne app IDs match console screenshot IDs (6380065f... and 95dc946f...) before committing. Do not use different IDs. |
| T-111-02 | **T**ampering | configStore.js | mitigate | Client IDs are public values (no secrets). Secrets stored only in env vars/KV (server-side only). |
| T-111-03 | **R**epudiation | agentMcpTokenService.js | mitigate | Log which client ID is being used for token exchange (already done at line 699 info level). Audit trail shows client choice. |
| T-111-04 | **I**nformation Disclosure | pingoneBackendDefaults.js | accept | Client IDs are public identifiers; not credentials. No PII or secrets. Low-value target for attackers. |
| T-111-05 | **D**enial of Service | configStore fallback chain | mitigate | If config retrieval fails, fallback to AGENT_OAUTH_CLIENT_ID (existing behavior). No service denial. |
| T-111-06 | **E**levation of Privilege | Token exchange | mitigate | Token exchange uses specified audience + scope narrowing. Correct client ID ensures scopes are properly enforced by PingOne. |
</threat_model>
