## Color Rules

| Rule | Detail |
|------|--------|
| **Dominant (60%)** | `--dash-bg` — page backgrounds, behind-card fill |
| **Secondary (30%)** | `--dash-surface` / `--dash-surface-2` — cards, nav, modal backgrounds |
| **Accent (10%)** | `--dash-accent` — active nav items, primary buttons, focus rings **only** |
| **Destructive** | `--dash-accent-red` — delete/remove actions only |

**Hardcoded value sweep rule:** Any `background: #fff`, `color: #000`, `color: #333`, `background: white`, `background-color: #ffffff` found on priority pages during implementation MUST be replaced with the appropriate CSS variable. No exceptions for in-scope files.

**Marketing path (D-2):** Remove the existing `App--marketing-page .main-content { background: white }` override from `App.css` (lines ~231–237). The `/marketing` route will inherit `html[data-theme='dark']` naturally. Add `html[data-theme='dark'] .landing-hero` background token to `LandingPage.css` if the hero gradient looks broken in dark mode.

## Component Contracts

### Theme Toggle Button (SideNav global toggle)

**Location:** `SideNav.js` — inside the `<div className="sidenav-footer">` block, **above** the Sign Out button.

**Collapsed state:** Show icon only (🌙 / ☀️), same behavior as other collapsed sidenav items.

**Expanded state:**
```jsx
<button
  type="button"
  className="sidenav-link sidenav-theme-toggle"
  onClick={toggleTheme}
  title={theme === 'dark' ? 'Switch to light mode' : 'Switch to dark mode'}
  aria-label="Toggle dark mode"
>
  <span className="sidenav-link-icon">{theme === 'dark' ? '☀️' : '🌙'}</span>
  {!collapsed && (
    <span className="sidenav-link-label">{theme === 'dark' ? 'Light mode' : 'Dark mode'}</span>
  )}
</button>
```

**CSS class:** `sidenav-theme-toggle` — inherits `.sidenav-link` base styles. Add to `SideNav.css`:
```css
/* Theme toggle — footer, above logout */
.sidenav-theme-toggle {
  /* Inherits sidenav-link styles */
  opacity: 0.85;
}
.sidenav-theme-toggle:hover {
  opacity: 1;
}
```

**Import:** `useTheme` from `'../context/ThemeContext'` — already the established pattern.

---

### Dark Mode CSS Override Pattern (all priority pages)

Add to each component's **own** `.css` file (not a shared override sheet). Pattern:

```css
/* Dark mode */
html[data-theme='dark'] .component-root {
  background: var(--dash-bg);
  color: var(--dash-text);
}
html[data-theme='dark'] .component-card {
  background: var(--dash-surface);
  border-color: var(--dash-border);
  box-shadow: var(--dash-shadow);
}
html[data-theme='dark'] .component-muted-text {
  color: var(--dash-muted);
}
```

**Selector scope:** Always prefix with `html[data-theme='dark']` — never `.dark` class or inline JS checks.

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Theme toggle click | Immediately applies `data-theme` to `document.documentElement`; no page reload |
| Dark mode persistence | Stored in `localStorage` + `sessionStorage` via `ThemeContext` (key: `banking_ui_theme`) |
| First paint | `public/index.html` inline script reads `localStorage` before React hydrates — no flash |
| Collapsed SideNav toggle | Icon-only button still fully keyboard accessible; `title` attribute provides tooltip |
| System preference | ThemeContext defaults to `'light'` regardless of `prefers-color-scheme` (existing behavior — do not change) |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Toggle keyboard accessible | `type="button"`, `aria-label="Toggle dark mode"` |
| Focus ring in dark mode | `--dash-accent` ring; ensure `outline: 2px solid var(--dash-accent)` on `:focus-visible` for new toggle |
| Color contrast | `--dash-text` (#e8edf5) on `--dash-surface` (#141a24) = ~14:1 — WCAG AAA |
| No Flash of Unstyled Content | Existing `index.html` inline script handles; do not remove it |
