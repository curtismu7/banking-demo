# Plan 04: Polish & Verification

**Objective:** Visual testing across all redesigned pages, fix any spacing/color inconsistencies, verify dark mode everywhere, and ensure all functionality intact.

**Context:** @113-CONTEXT.md

---

## Tasks

### Task 1: Cross-page visual audit

**Description:** Manual testing of LandingPage, UserDashboard, Dashboard, and admin subpages. Verify Chase colors consistent, typography clean, no layout breakage.

**Checklist:**
- [ ] LandingPage hero: navy background, white headline, styled CTAs work
- [ ] LandingPage features section: adequate contrast, readable
- [ ] UserDashboard: ChaseTopNav visible, logout button works, user email shows
- [ ] UserDashboard: tables/cards styled with Chase colors, no color clashes
- [ ] Dashboard (admin): ChaseTopNav visible, admin actions intact
- [ ] Accounts, Transactions, Users pages: white cards, gray borders, navy headers
- [ ] AuditPage: table headers light gray, badges readable
- [ ] FeatureFlagsPage: toggles styled, badges visible
- [ ] All pages: `npm run build` passes, no console errors

### Task 2: Dark mode verification

**Description:** Test `html[data-theme='dark']` rendering on all pages.

**Checklist:**
- [ ] Click theme toggle on any page
- [ ] LandingPage hero inverts correctly (dark surface bg, light text)
- [ ] UserDashboard dark: tables readable, text has contrast
- [ ] Dashboard (admin) dark: nav readable, text visible
- [ ] Admin pages dark: badges/status colors still distinct
- [ ] No hardcoded colors break dark mode (should be none left)
- [ ] Dark mode toggle persists across page navigations

### Task 3: Responsive design check

**Description:** Test pages at common breakpoints (mobile, tablet, desktop).

**Checklist:**
- [ ] ChaseTopNav remains horizontal and readable on mobile
- [ ] Tables don't overflow on mobile (scroll or stack as appropriate)
- [ ] Hero section still prominent on mobile
- [ ] CTA buttons remain clickable (not too small)
- [ ] Font sizes readable across all widths

### Task 4: Functionality verification

**Description:** Confirm all interactive elements still work after styling changes.

**Checklist:**
- [ ] Logout button works (both pages with ChaseTopNav)
- [ ] Theme toggle works and persists
- [ ] Navigation links work (if any internal routing)
- [ ] Admin role switch button works (if applicable)
- [ ] Table sorting/filtering works (if present in admin pages)
- [ ] Modals/dialogs don't have color clashes on top of new styling
- [ ] Disabled button states still visually distinct

### Task 5: Optional: Mobile hamburger menu

**Description:** If ChaseTopNav is too crowded on mobile, add a hamburger menu (optional — can be deferred to next phase).

**Output:** `.chase-top-nav__hamburger` styled toggle + menu animation

**Implementation notes:**
- Only add if `npm start` testing reveals mobile nav is cramped
- If added: commit separately as hotfix or include in polish commit

### Task 6: Final polish commit

**Description:** Make final tweaks (spacing adjustments, color fine-tuning, etc.) and commit.

**Potential tweaks:**
- Adjust nav bar shadow if too harsh or too subtle
- Fine-tune button hover opacity or transitions
- Ensure badge colors match Chase conventions exactly
- Fix any orphaned or inconsistent spacing
