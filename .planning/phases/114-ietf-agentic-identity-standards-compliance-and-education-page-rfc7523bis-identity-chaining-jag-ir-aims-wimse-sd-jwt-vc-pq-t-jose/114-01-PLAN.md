**Action:**
Create a new React functional component following the BestPracticesPanel pattern. Component structure:

1. **Imports:** React hooks (useState), styling, any icon/badge dependencies from existing panels
2. **Tab definitions:** 8 tabs (one per requirement):
   - `overview` — IDC guardrail table (5 rows) + "How this demo implements each guardrail" intro
   - `rfc7523bis` — RFC7523bis JWT Client Auth update
   - `identity-chaining` — Identity Chaining + ID-JAG cross-domain delegation
   - `jag-ir` — JWT Grant Interaction Response (HITL for agents)

3. **Each tab content** (copy verbatim from CONTEXT.md):

   - What it defines
   - Working Group (OAuth WG / WIMSE WG / JOSE WG / Network WG)
   - Maturity level (Very High / High / Medium-High / Medium / Early)
   - Ping Identity role (all: Co-author)
   - Current demo implementation status (✅ Full / ⚠️ Partial / ❌ Not implemented)
   - Gap description
   - IETF draft link

4. **Overview tab content** (copy from Phase 114 CONTEXT.md):
   - Table: 5 IDC guardrails × 7 standards mapping
   - For each guardrail: which standard(s) fulfill it, current implementation status
   - Summary paragraph: "How this demo implements each guardrail"

5. **Maturity badge styling:**

   - High / Current: Blue (#2563eb)
   - Medium-High: Orange (#d97706)
   - Medium / Early: Yellow (#ca8a04)

6. **Component exports:** Export as default function `IETFStandardsPanel({ onClose })`
