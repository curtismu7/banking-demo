## RFC 8693 Token Exchange (Current Pattern)

From existing codebase (`CLAUDE.md` and `oauth-pingone` skill):
- OAuth 2.0 Authorization Code + PKCE flow for user login
- User receives access_token + refresh_token
- For agent operations: RFC 8693 token exchange
  - User (actor) exchanges their token for agent-scoped token
  - `act` claim identifies the agent
  - `may_act` claim in user's token authorizes delegation
- Token events logged for UI display: exchange token, get events

## Session Management (Current)

From existing code:
- Session store: Upstash Redis (Vercel-compatible)
- Express-session middleware on BFF
- Session refresh on OAuth 401
- Cookies: httpOnly, secure, SameSite=Strict

## Integration Points

1. Agent executor needs user context (email, user_id, session)
2. MCP tool calls must use agent-scoped token (RFC 8693)
3. Token events from tool calls must be bubbled up for UI
4. Session refresh must happen transparently

 * Agent Session Middleware
 * Binds agent executor to authenticated user context
 * Handles RFC 8693 token exchange + session refresh

export async function agentSessionMiddleware(req, res, next) {
  // Verify user is authenticated
  if (!req.user || !req.session) {
    return res.status(401).json({ error: 'Unauthorized. Please login.' });
  }

  // Validate session still valid (optional: check expiry)
  if (req.session.expiresAt && req.session.expiresAt < Date.now()) {
    try {
      await refreshOAuthSession(req);
    } catch (error) {
      return res.status(401).json({ error: 'Session expired. Please login again.' });
    }
  }

  // Attach auth context to request for agent service
  req.agentContext = {
    userId: req.user.sub,  // PingOne user ID
    email: req.user.email,
    accessToken: req.session.oauth_tokens.access_token,
    refreshToken: req.session.oauth_tokens.refresh_token,
    sessionId: req.sessionID,
  };

  // Initialize token events tracking for this request
  req.tokenEvents = [];

  next();
}

// Import existing services for token exchange + RFC 8693
import { callMcpTool, refreshOAuthSession } from './bankingAgentService';

/**
 * Extended: Process banking agent message with auth context
 * Handles RFC 8693 token exchange + token event tracking
 */
export async function processBankingAgentMessage(
  message,
  executor,
  userId,
  agentContext,  // NEW: from middleware
  tokenEvents    // NEW: from middleware
) {
  try {
    // Step 1: Exchange user token for agent token (RFC 8693)
    const agentToken = await exchangeTokenForAgent(agentContext, tokenEvents);

    // Step 2: Create tool wrapper that injects agent token
    const toolsWithContext = createMcpToolRegistry().map(tool => {
      return {
        ...tool,
        // Override _call to use agent token + track events
        _call: async (input) => {
          try {
            const result = await callMcpTool(
              tool.name,
              input,
              agentToken,  // Pass agent token
              agentContext.userId,
              tokenEvents  // Track exchange events
            );
            return JSON.stringify(result);
          } catch (error) {
            return JSON.stringify({ error: error.message });
          }
        },
      };
    });

    // Step 3: Invoke agent with context
    const response = await executor.invoke({
      input: message,
      userId,
      context: agentContext,
    });

    // Step 4: Return response + token events
    return {
      success: true,
      message: response.output,
      tokenEvents: tokenEvents,  // Return for UI display
    };
  } catch (error) {
    console.error('[bankingAgentLangChain] Agent error:', error);
    return {
      success: false,
      error: error.message || 'Agent error',
      tokenEvents: tokenEvents,  // Return partial events even on error
    };
  }
}

/**
 * RFC 8693 Token Exchange
 * User acts on behalf of agent
 */
async function exchangeTokenForAgent(agentContext, tokenEvents) {
  try {
    // Call PingOne token endpoint with RFC 8693 params
    // subject_token = user's access token
    // actor_token = (optional, if agent has credentials)
    // requested_token_use = "access_token"
    
    const response = await fetch(process.env.PINGONE_TOKEN_ENDPOINT, {
      method: 'POST',
      headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
      body: new URLSearchParams({
        grant_type: 'urn:ietf:params:oauth:grant-type:token-exchange',
        subject_token: agentContext.accessToken,
        subject_token_type: 'urn:ietf:params:oauth:token-type:access_token',
        requested_token_use: 'access_token',
        act: JSON.stringify({ sub: 'banking-agent', name: 'BankingAgent' }),
        client_id: process.env.OAUTH_CLIENT_ID,
        client_secret: process.env.OAUTH_CLIENT_SECRET,
      }),
    });

    const data = await response.json();
    if (!response.ok) {
      throw new Error(data.error_description || 'Token exchange failed');
    }

    // Track token exchange event
    tokenEvents.push({
      type: 'token_exchange',
      timestamp: new Date().toISOString(),
      from: 'user',
      to: 'agent',
      actor: { sub: 'banking-agent' },
    });

    return data.access_token;
  } catch (error) {
    console.error('[tokenExchange] RFC 8693 error:', error);
    throw error;
  }
}
```

/**
 * Call MCP tool with agent token + event tracking
 * Records token events for UI + auditing
 */
async function callMcpTool(
  toolName,
  params,
  agentToken,       // NEW: RFC 8693 token
  userId,           // NEW: user context
  tokenEvents = []  // NEW: event log
) {
  try {
    // Use agent token for authorization
    const response = await fetch(`${process.env.BFF_URL}/api/mcp/tool`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${agentToken}`,  // Use agent token
        'X-User-Id': userId,  // Track user performing action
      },
      body: JSON.stringify({
        tool: toolName,
        params: params,
      }),
    });

    const data = await response.json();

    // Track tool invocation event
    tokenEvents.push({
      type: 'tool_call',
      timestamp: new Date().toISOString(),
      tool: toolName,
      status: response.ok ? 'success' : 'failed',
      actor: 'agent',
      onBehalfOf: userId,
    });

    if (!response.ok) {
      // Handle 401: refresh token + retry
      if (response.status === 401) {
        // Trigger refresh (depends on session middleware)
        throw new Error('Unauthorized: token refresh needed');
      }
      throw new Error(data.error || 'MCP tool failed');
    }

    return data.result || data;
  } catch (error) {
    tokenEvents.push({
      type: 'tool_error',
      timestamp: new Date().toISOString(),
      tool: toolName,
      error: error.message,
    });
    throw error;
  }
}
```

**Key requirements:**
- Accept agentToken (from RFC 8693 exchange)
- Use agent token in Authorization header
- Track userId in X-User-Id header (audit trail)
- Log tool_call and tool_error events
- Return successful results + event log
- Handle 401 gracefully

**Backwards compatibility:**
- Old signature callMcpTool(tool, params) should still work (add defaults)
- New callers use callMcpTool(tool, params, agentToken, userId, tokenEvents)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → Agent | User acts on behalf of agent via RFC 8693 — delegation is explicit and auditable |
| Agent → MCP Tools | Agent uses delegated token — tools can verify actor claims |
| Session | Session tokens stored in Upstash Redis — secure, stateless, Vercel-compatible |
| Token Events | Logged to session context — replayed to UI, never stored in database |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-115-02-01 | Spoofing | User impersonates another user | mitigate | OAuth 2.0 ensures authentication; PingOne validates identity; session bound to user |
| T-115-02-02 | Tampering | Token exchanged but actor claim forged | mitigate | RFC 8693 actor claim is cryptographically bound in JWT; MCP tools verify actor claim |
| T-115-02-03 | Elevation of Privilege | Agent calls tools beyond its scope | mitigate | Agent token only valid for delegated operations; MCP tools enforce scope/HITL |
| T-115-02-04 | Information Disclosure | Token events leaked in logs | mitigate | Token events logged to session context only (not persisted); sanitized before UI display |
| T-115-02-05 | Denial of Service | Token exchange exhausts quota | mitigate | Rate limit on token endpoint; log exchange count; circuit breaker if excessive |

</threat_model>
