**File: banking_api_server/routes/mcpDiscoveryRoutes.js**

Requirements:
1. Import express, rateLimit, createMcpToolRegistry from utils/mcpToolRegistry
2. Define PUBLIC_TOOLS constant: ['explain_topic', 'brave_search']
3. Create discoveryLimiter middleware:
   - windowMs: 60 * 1000 (1 minute)
   - max: 100 requests per window
   - keyGenerator: (req) => req.ip (IP-based)
   - skip: () => process.env.DISABLE_RATE_LIMIT === 'true'
   - handler: Return 429 with JSON error + retryAfter header
4. Create getPublicToolsList() function:
   - Calls createMcpToolRegistry()
   - Filters to only PUBLIC_TOOLS by name
   - Returns array of tool objects
5. Create toolToJsonSchema(tool) converter:
   - Converts Zod schema to JSON Schema format
   - Returns { name, description, schema } object suitable for JSON response
6. Register two GET routes:
   - GET / → responds with JSON: { server: { name, version, ... }, tools: [...] }
   - Both routes use discoveryLimiter middleware
7. Export router as default

Response schema (both endpoints):
```json

  "server": {
    "name": "Banking MCP Server",
    "version": "1.0.0",
    "description": "MCP tools for banking operations and education"

  "tools": [
    {
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Unauthenticated → Tool Whitelist | Discovery endpoint exposes only safe tools; rates limited to prevent DoS |
| Rate Limiter → Backend | IP-based rate limiting prevents single client from overloading discovery |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-119-01 | Spoofing | Rate limiter (IP-based) | mitigate | Express-rate-limit uses `req.ip` directly; behind reverse proxy must trust X-Forwarded-For (handled by Helmet) |
| T-119-02 | Information Disclosure | Tool whitelist filter | mitigate | get_login_activity kept behind auth (not in PUBLIC_TOOLS); only `explain_topic` and `brave_search` exposed |
| T-119-03 | Denial of Service | Discovery endpoint | mitigate | Rate limit 100 req/min per IP prevents request flooding; separate limiter for discovery vs global |
| T-119-04 | Tampering | Tool list serialization | mitigate | Fetch fresh tool registry on each request (no caching of tool list); tool metadata comes from createMcpToolRegistry() directly |

</threat_model>
