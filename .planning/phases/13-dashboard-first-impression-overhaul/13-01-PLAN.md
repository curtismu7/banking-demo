Create hero styling using the CSS tokens defined in Task 1. Requirements:

**Desktop layout (≥1024px):**
- Hero container: full width, 40% of above-fold (estimate 280-320px height including padding)
- Background: modern gradient or primary color with opacity overlay (use --color-primary-light or subtle gradient)
- Content area: centered horizontally, max-width container (1200px typical)
- Hero text and actions: flex row, side-by-side
- CTAs: prominent buttons with primary/secondary styling

**Mobile layout (<1024px):**
- Hero container: full width, slightly reduced height
- Content area: padding adjustments for mobile (use --spacing-md)
- Hero text: stack vertically with actions below
- CTAs: full-width or stacked buttons for touch targets
- Font sizes: scale down using --font-size-lg instead of --font-size-2xl

**Styling details:**
- Use design tokens for colors, spacing, shadows
- `.hero-container` — outer section with background
- `.hero-content` — inner flex container
- `.hero-text` — contains h1 and p
- `.hero-title` — h1 styled with --font-size-2xl, --color-text-primary, font-weight 600+
- `.hero-subtitle` — p styled with --font-size-md, --color-text-secondary, line-height 1.5+
- `.hero-actions` — flex row on desktop, flex column on mobile
- `.hero-cta` — base button styling (padding, border-radius, transition)
- `.hero-cta-primary` — primary color background, white text
- `.hero-cta-secondary` — border style, primary color border, transparent background
- `.hero-education-chips` — placeholder area, initially invisible or minimal height

- Mobile-first approach: base styles for mobile
- `@media (min-width: 768px)` — tablet
- `@media (min-width: 1024px)` — desktop

**Performance notes:**
- Minimal animations (avoid heavy reflows)
- No custom fonts that need download (use system font stack or already-available fonts)
- Leverage browser's native rendering
