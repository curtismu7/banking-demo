Create AccountCard component for individual account display. Requirements:

**Component structure:**
- Card container with shadow (--shadow-md)
- Account name/identifier at top (e.g., "Checking", "Savings")
- Large balance figure prominently (primary focus of card)
- Currency indicator
- Account status badge (active/inactive)
- Quick actions row (micro-buttons for "View" or "Transfer")

**Props:**
- `account` object: { id, name, balance, currency, status }
- `onSelectAccount` — callback when card clicked
- `onQuickAction` — callback for action buttons

**Code pattern:**
```jsx
import React from 'react';

export const AccountCard = ({ account, onSelectAccount, onQuickAction }) => {
  const { id, name, balance, currency = 'USD', status = 'active' } = account;
  
  return (
    <div className="account-card" onClick={() => onSelectAccount?.(id)}>
      <div className="account-card-header">
        <h3 className="account-name">{name}</h3>
        <span className={`account-status account-status-${status}`}>
          {status}
        </span>
      </div>
      
      <div className="account-card-body">
        <div className="balance-display">
          <span className="balance-label">Balance</span>
          <span className="balance-amount">
            {currency} {balance?.toLocaleString('en-US', { minimumFractionDigits: 2 })}
          </span>
        </div>
      </div>
      
      <div className="account-card-actions">
        <button 
          className="card-action-button"
          onClick={(e) => {
            e.stopPropagation();
            onQuickAction?.('view', id);
          }}
        >
          View
        </button>
        <button 
          className="card-action-button"
          onClick={(e) => {
            e.stopPropagation();
            onQuickAction?.('transfer', id);
          }}
        >
          Transfer
        </button>
      </div>
    </div>
  );
};

export default AccountCard;
```

**Accessibility:**
- Semantic HTML using `<div role="button">` or button elements
- ARIA labels if not text-labeled
- Keyboard navigation (clickable with Enter key)
- Color contrast ≥4.5:1 for balance text

**Styling expectations:**
- Uses tokens: --color-primary, --color-text-primary, --color-success/warning/error (for status colors)
- Card styling with --shadow-md, --border-radius
- Balance figure in larger font (--font-size-xl or --font-size-2xl)

Create AccountSummary component that wraps AccountCard components in a responsive grid. Requirements:

**Component structure:**
- Grid container (responsive: 1 column mobile, 2-3 columns desktop)
- Maps over `accounts` array and renders AccountCard for each
- Empty state message if no accounts
- Total assets summary at bottom (optional)

**Props:**
- `accounts` — array of account objects
- `onSelectAccount` — passed through to AccountCard
- `onQuickAction` — passed through to AccountCard

**Code pattern:**
```jsx
import React from 'react';
import AccountCard from './AccountCard';

export const AccountSummary = ({ accounts = [], onSelectAccount, onQuickAction }) => {
  if (!accounts || accounts.length === 0) {
    return (
      <div className="account-summary">
        <p className="empty-state">No accounts available</p>
      </div>
    );
  }

  const totalBalance = accounts.reduce((sum, acc) => sum + (acc.balance || 0), 0);

  return (
    <div className="account-summary">
      <h2 className="summary-title">Your Accounts</h2>
      
      <div className="account-cards-grid">
        {accounts.map(account => (
          <AccountCard
            key={account.id}
            account={account}
            onSelectAccount={onSelectAccount}
            onQuickAction={onQuickAction}
          />
        ))}
      </div>
      
      <div className="total-assets">
        <span>Total Assets:</span>
        <span className="total-balance">${totalBalance.toLocaleString('en-US', { minimumFractionDigits: 2 })}</span>
      </div>
    </div>
  );
};

export default AccountSummary;
```

**Styling expectations:**
- Grid layout with CSS Grid or Flexbox (responsive)
- Uses --spacing-* tokens for gaps and padding
- Responsive breakpoints: 1 column mobile, 2 columns tablet, 3 columns desktop

Create ActionHub component for primary actions (send money, request funds, deposit, etc.). Requirements:

**Component structure:**
- Action hub container (25% of above-fold from CONTEXT.md)
- Grid or flex layout of action buttons
- Primary actions: Send, Request, Deposit, Withdraw (per banking demo capabilities)
- Context-aware display (disable some actions if no account selected)
- Integration point for Agent action

**Props:**
- `selectedAccountId` — context for which account actions apply to
- `onAction` — callback(actionType, payload) when action clicked
- `enabled` — whether actions are available (false during loading)

**Code pattern:**
```jsx
import React from 'react';
import './ActionHub.css';

const ACTIONS = [
  { id: 'send', label: 'Send Money', icon: '→', color: 'primary' },
  { id: 'request', label: 'Request', icon: '←', color: 'secondary' },
  { id: 'deposit', label: 'Deposit', icon: '+', color: 'success' },
  { id: 'withdraw', label: 'Withdraw', icon: '-', color: 'warning' },
];

export const ActionHub = ({ selectedAccountId, onAction, enabled = true }) => {
  return (
    <section className="action-hub">
      <h2 className="action-hub-title">Quick Actions</h2>
      
      <div className="action-buttons-grid">
        {ACTIONS.map(action => (
          <button
            key={action.id}
            className={`action-button action-button-${action.color}`}
            disabled={!enabled || !selectedAccountId}
            onClick={() => onAction?.(action.id, { accountId: selectedAccountId })}
            aria-label={action.label}
          >
            <span className="action-icon">{action.icon}</span>
            <span className="action-label">{action.label}</span>
          </button>
        ))}
      </div>
      
      {!selectedAccountId && (
        <p className="action-hub-hint">Select an account to enable actions</p>
      )}
    </section>
  );
};

export default ActionHub;
```

**Accessibility:**
- Button elements with aria-label
- Disabled state properly communicated
- Visually distinct primary/secondary buttons
- Keyboard navigation (Tab between buttons)

**Styling expectations:**
- Uses design tokens: --color-primary, --color-success, --color-warning
- Buttons with padding (--spacing-md), border-radius (--border-radius)
- Hover states, active states, disabled states
- Responsive: 2-4 buttons per row depending on screen size
