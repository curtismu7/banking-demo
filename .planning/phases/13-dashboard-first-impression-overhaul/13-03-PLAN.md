Create a reusable SkeletonLoader component to show structure while data loads. Requirements:

**Component structure:**
- Generic skeleton shape (rectangle, circle, text line)
- Animated shimmer or pulse effect
- Configurable width, height, count (for rendering multiple skeletons)
- Support for different skeleton types (card, text, circle for avatars)

**Props:**
- `type` — 'card' | 'text' | 'circle' | 'list'
- `count` — number of skeleton items to render (default 1)
- `width` — CSS width (default 100%)
- `height` — CSS height (default varies by type)

**Code pattern:**
```jsx
import React from 'react';
import './SkeletonLoader.css';

export const SkeletonLoader = ({ 
  type = 'card', 
  count = 1, 
  width = '100%', 
  height = undefined 
}) => {
  const renderSkeleton = (key) => {
    switch (type) {
      case 'circle':
        return (
          <div 
            key={key}
            className="skeleton skeleton-circle"
            style={{ width: height || '40px', height: height || '40px' }}
          />
        );
      case 'text':
        return (
          <div 
            key={key}
            className="skeleton skeleton-text"
            style={{ width, height: height || '14px', marginBottom: '8px' }}
          />
        );
      case 'card':
      default:
        return (
          <div 
            key={key}
            className="skeleton skeleton-card"
            style={{ width, height: height || '150px' }}
          />
        );
    }
  };

  return (
    <div className="skeleton-loader">
      {Array.from({ length: count }).map((_, i) => renderSkeleton(i))}
    </div>
  );
};

export default SkeletonLoader;
```

**Animation expectations:**
- Smooth shimmer or pulse effect (50-100ms per cycle)
- Uses design tokens for colors (--color-* variables)
- No jarring flashes or performance issues

**Accessibility:**
- aria-busy="true" on loading skeletons
- Hidden from screen readers (aria-hidden or similar)

  <name>Task 2: Create useDashboardData custom hook for data fetching</name>
  <files>banking_api_ui/src/hooks/useDashboardData.js</files>
  <action>
Create a custom React hook for optimized dashboard data fetching. Requirements:

**Hook behavior:**
- Fetches accounts and recent transactions on mount
- Manages loading, error, and data states
- Provides refetch capability
- Caches data to avoid redundant fetches
- Handles errors gracefully

**Hook signature:**
```javascript
const { data, loading, error, refetch } = useDashboardData()
```

Where:
- `data` — { accounts: [], transactions: [] }
- `loading` — boolean
- `error` — Error object or null
- `refetch` — function to manually refresh data

**Implementation pattern:**
```jsx
import { useState, useEffect, useCallback } from 'react';

export const useDashboardData = () => {
  const [data, setData] = useState({ accounts: [], transactions: [] });
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  const [lastFetch, setLastFetch] = useState(0);

  const fetchData = useCallback(async () => {
    try {
      setLoading(true);
      setError(null);

      // Fetch accounts
      const accountsRes = await fetch('/api/accounts/my');
      if (!accountsRes.ok) throw new Error(`Failed to fetch accounts: ${accountsRes.status}`);
      const accounts = await accountsRes.json();

      // Fetch recent transactions
      const transRes = await fetch('/api/transactions/my?limit=10');
      if (!transRes.ok) throw new Error(`Failed to fetch transactions: ${transRes.status}`);
      const transactions = await transRes.json();

      setData({ accounts, transactions });
      setLastFetch(Date.now());
    } catch (err) {
      setError(err);
      console.error('Dashboard data fetch error:', err);
    } finally {
      setLoading(false);
    }
  }, []);

  useEffect(() => {
    const shouldFetch = Date.now() - lastFetch > 60000; // Refetch every 60s
    if (shouldFetch || lastFetch === 0) {
      fetchData();
    }
  }, [fetchData, lastFetch]);

  return { data, loading, error, refetch: fetchData };
};
```

**Performance considerations:**
- Caching: don't refetch more than every 60 seconds
- Parallel requests: fetch accounts and transactions simultaneously (Promise.all)
- Error handling: individual fetch failures don't block other data

Create an ErrorBoundary component to gracefully handle errors in dashboard and child components. Requirements:

**Component pattern (React ErrorBoundary pattern):**
```jsx
import React from 'react';

export class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, error: null };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  componentDidCatch(error, errorInfo) {
    console.error('Dashboard error:', error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return (
        <div className="error-boundary">
          <div className="error-message">
            <h2>Something went wrong</h2>
            <p>{this.state.error?.message || 'An unexpected error occurred'}</p>
            <button onClick={() => window.location.reload()}>
              Reload Page
            </button>
          </div>
        </div>
      );
    }

    return this.props.children;
  }
}

export default ErrorBoundary;
```

**Error scenarios handled:**
- Component render errors
- State update errors
- Event handler errors (if caught and re-thrown)

**Does NOT handle:**
- Async errors (use try/catch in hooks/callbacks)
- Event listener errors (address separately)

**Styling for error display:**
- Clear error message
- Reload button (full-width or centered)
- Use --color-error token for error styling
