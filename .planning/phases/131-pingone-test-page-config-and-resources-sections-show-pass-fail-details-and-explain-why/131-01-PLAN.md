<objective>
Enhance the Configuration and Resources sections of PingOneTestPage so each TestCard shows:
1. The environment variable name that controls the value (e.g. `PINGONE_ENVIRONMENT_ID`)
2. A format hint (e.g. `UUID — e.g. 12345678-1234-1234-1234-123456789012`)
3. When failed: an inline amber box with the exact fix instruction (what to add to .env)
4. When passed: env var + format hint still visible as reference context

Purpose: Operators can now understand at-a-glance why each item passes or fails and what to do about it — without needing to click "Fix in PingOne ↗" and read a disappearing toast.

Output: Updated PingOneTestPage.jsx and PingOneTestPage.css. No backend changes.
</objective>
