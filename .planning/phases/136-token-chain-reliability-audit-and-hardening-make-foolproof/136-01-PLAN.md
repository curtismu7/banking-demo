<objective>
Fix the root cause why the token chain is always empty and shows placeholder:
- **Bug**: `oauth.js` stores events with `authedUser.id` (internal DB integer/UUID) but `/api/token-chain` reads by `req.user.id = decoded.sub` (PingOne UUID) — key mismatch means `tokenEvents.get(userId)` always returns `[]`.
- **Secondary**: in-memory Map resets on server restart, so chain is always empty after restart.

Fix by:
1. Changing `oauth.js` to use `decoded.sub` as the userId key (consistent with oauthService.js).
2. Adding a `synthesizeFromSession` fallback in the GET `/api/token-chain` route: if Map returns empty but session has an access token, decode it and return a synthetic auth event.

Purpose: After this plan, `/api/token-chain` will return real data for every logged-in user regardless of server restarts.
Output: Fixed `oauth.js`, updated server route with synthesis fallback, new `synthesizeFromSession` export.
</objective>

**In `tokenChainService.js`** — add a new exported function after `clearTokenChain`:

```js
/**
 * Synthesize a single auth event from a raw access token (fallback for cold-start / restart).
 * Called by the tokenChain route when the in-memory Map has no events for the user.
 * Returns an array with one synthetic event, or [] if token cannot be decoded.
 */
function synthesizeFromSession(accessToken) {
  if (!accessToken || typeof accessToken !== 'string') return [];
  try {
    const claims = jwt.decode(accessToken);
    if (!claims?.sub) return [];
    return [{
      id: 'synthetic-session-' + claims.sub.slice(0, 8),
      timestamp: new Date().toISOString(),
      eventType: 'auth',
      tokenType: 'user_token',
      tokenSub: claims.sub,
      tokenAct: claims.act || null,
      tokenAgent: claims.act?.client_id || null,
      scopes: claims.scope ? (Array.isArray(claims.scope) ? claims.scope : claims.scope.split(' ')) : [],
      audience: Array.isArray(claims.aud) ? claims.aud.join(' ') : (claims.aud || ''),
      issuer: claims.iss || '',
      expiry: claims.exp ? new Date(claims.exp * 1000).toISOString() : null,
      description: 'User session token (synthesized from active BFF session — server restarted or cold start)',
      exchangeSteps: [],
      userId: claims.sub,
      _synthetic: true,
    }];
  } catch (_e) { return []; }
}
```

Add `synthesizeFromSession` to the `module.exports` object.
