<objective>
Fix all broken flows on both test pages so every button does something correct.

**PingOne Test fixes:**
1. Token exchange (single + double) — currently passes `workerToken` string `'valid'` as JWT subject. Fix: pass `authzToken` (the decoded authz JWT from state) as `subjectToken`, and `agentToken` as `actorToken` for double exchange. Disable exchange buttons if prerequisite token not yet fetched.
2. The generic `/api/pingone-test/token-exchange` backend endpoint already calls `oauthService.performTokenExchange(subjectToken, ...)` — it just needs a real JWT from frontend.
3. Verify all 6 test card Fix buttons open the correct PingOne console URL.

**MFA Test fixes:**
1. FIDO2 initiate — currently calls backend but `navigator.credentials.create()` is never invoked. Add real WebAuthn call using the challenge from backend.
2. Device list — ensure response maps to correct display fields.
3. All section Test buttons return meaningful pass/fail with error detail.
</objective>

<execution_steps>
## Step 1 — PingOne Test: fix exchange subjectToken

In `PingOneTestPage.jsx`:
- Add state `const [authzToken, setAuthzToken] = useState(null)` (raw JWT string from authz-token response)
- In existing authz-token load callback, store `data.rawToken` (backend must return it — see Step 1b)
- Single exchange: `subjectToken: authzToken`, disabled when `!authzToken`
- Double exchange: `subjectToken: authzToken, actorToken: agentTokenRaw` (raw agent JWT from agent-token response)

In `pingoneTestRoutes.js` authz-token route:
- Add `rawToken: authzToken.substring(0, 40) + '...'` (preview only) to response
- Add `tokenId` (jti claim) to response for display

NOTE: Do NOT return the full raw JWT to browser. The exchange backend route will call `oauthService.performTokenExchange` using the session's authz token internally. The frontend should trigger the exchange by calling a session-scoped endpoint, not by passing raw tokens.

**Correct approach** (BFF pattern — tokens stay server-side):
- Add `POST /api/pingone-test/exchange-from-session` endpoint:
  - `mode`: `'single'` or `'double'`
  - Backend reads `req.session.oauthTokens.accessToken` as subjectToken
  - Backend reads agent client credentials token as actorToken (for double)
  - Returns decoded exchanged token (no raw JWT to browser)
- Frontend calls this instead of the generic `/token-exchange`

## Step 2 — MFA Test: FIDO2 real WebAuthn

In `MFATestPage.jsx` fido2 section:
- After `POST /api/mfa/test/fido2-init` returns `{ challenge, rpId, userId }`:
  - Call `navigator.credentials.create({ publicKey: { challenge, rp, user, ... } })`
  - Send result to new backend endpoint `POST /api/mfa/test/fido2-complete`
  - Show pass/fail with credential ID display

In `mfaTest.js`:
- Add `POST /api/mfa/test/fido2-complete` route that calls PingOne FIDO2 verification endpoint
