<objective>
Make the PingOne entity mapper on /pingone-test the single source of truth for what's configured in PingOne — as comprehensive as a dashboard.

**Backend enrichment** (`GET /api/pingone-test/verify-assets`):

### New data collected (all using existing worker/management token):

1. **Users** (`GET /v1/environments/{envId}/users?limit=20`):
   - `sub` (id), `username`, `email`, `population.id`, `lifecycle.status`
   - Custom attributes: any `p1.` prefixed attributes

2. **SPEL / Token Policies** (`GET /v1/environments/{envId}/tokenPolicies`):
   - For each app, retrieve associated token policy
   - Extract `claimsMapper` → SPEL expressions if present
   - Extract `tokenIntrospectionEndpointAuthMethod`

3. **Resource audiences** — each resource server already fetched; add `audience` field from the resource object

4. **Per-app resource grant matrix** — already partially built in Phase 130; complete it:
   - For each app: `GET /v1/environments/{envId}/applications/{appId}/grants`
   - Returns `[ { resource: { id, name }, scopes: [ { scope: { id, name } } ] } ]`
   - Map to: `{ appId, appName, grants: [ { resourceId, resourceName, scopes: [scopeName] } ] }`

**Frontend tabs** (`AssetTable.js`):

| Tab | Content |
|---|---|
| Apps | App name, type, clientId, grant types, status |
| Resources | Resource name, audience URI, scope count |
| Scopes | Scope name, resource, description |
| Users | Username, email, population, status, custom attrs |
| SPEL | App name, policy name, SPEL expression(s) |
| Grants | App → Resource → Scopes matrix (heat map rows) |

Each tab has: count badge, pass/fail icon (green if data present, amber if empty, red if error).
</objective>

<execution_steps>
## Step 1 — Backend: users endpoint

In `pingoneTestRoutes.js` verify-assets handler, after existing asset calls, add:
```js
const usersResp = await managementService.getUsers({ limit: 20 });
assets.users = { status: 'passed', count: usersResp.length, data: usersResp };
```
Implement `managementService.getUsers()` if not present (call `GET /v1/environments/{envId}/users?limit=20`).

## Step 2 — Backend: SPEL / token policies

Add call: `GET /v1/environments/{envId}/tokenPolicies`
Map each policy to `{ policyId, name, appId, spelExpressions: [] }`.
If PingOne doesn't expose SPEL this way, fall back to showing the claim mapper rules.

## Step 3 — Backend: per-app resource grant matrix (complete)

For each application in the apps list:
- `GET /v1/environments/{envId}/applications/{appId}/grants`
- Parse into `{ appId, appName, grants: [{ resourceId, resourceName, scopes }] }`
- Add `grantMatrix` to verify-assets response

## Step 4 — Frontend: AssetTable tabs

Update `AssetTable.js` to render tabs: Apps | Resources | Scopes | Users | SPEL | Grants
Each tab reads from the enriched `verify-assets` response.
Tab header shows count badge and pass/fail color.

## Step 5 — Build verify

`cd banking_api_ui && npm run build` exits 0.
</execution_steps>
