# Phase 14 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Problem Statement

Two concrete problems observed in the floating agent window:

**Problem 1 — Left rail is visually cluttered**
The 130px sidebar shows 4 section labels ("SESSION", "Try asking:", "Actions:", "Learn & Explore:")
and 8+ interactive items stacked in a small column. Section labels compete with button labels and
the result looks like a navigation sidebar crammed into a chat widget.

**Problem 2 — Agent panel covers dashboard content**
The floating panel is positioned at `right: 192px`. This offset was historically left over from a
time when the FAB pill was visible while the panel was open. Today the FAB is hidden when the panel
is open, so there is no reason for the 192px offset. It causes the 400px-wide panel to land in the
center of the screen, covering the dashboard's account/transaction table.

---

## Scope

**In scope (CSS and minimal JSX only):**
- Left rail: remove section label noise, collapse SESSION actions behind the refresh icon button
- Panel position: move float anchor from `right: 192px` to `right: 16px`
- Panel width: reduce float default from 400px to 380px (minor — main fix is position)
- "Learn & Explore" section: collapse to a single `▾` toggle button that expands inline

**Out of scope (do NOT change):**
- BankingAgent.js logic, state, hooks, or event handlers
- Any token exchange / OAuth flows
- Inline/embedded/split3 layout modes — only float mode is touched
- The right column (chat messages + input)
- Any server-side code

## Layout Contract

### Floating Panel Position

| Property | Before | After |
|----------|--------|-------|
| `right` | `192px` | `16px` |
| `width` (default) | `400px` | `380px` |
| `min-width` | `280px` | `280px` (unchanged) |

**Rationale:** FAB is hidden while panel is open. The 192px offset has no purpose. Anchoring to
`right: 16px` keeps the panel in the bottom-right corner, matching the FAB position users expect,
and stops it from covering the dashboard's account table.

### Left Rail Structure (float mode)

Before (4 labeled sections, 8+ items):
```
SESSION
  [🔄 Refresh access token]
  [🔐 Sign in again]

TRY ASKING:
  ["Show me my accounts"]
  ["Transfer $100 to savings"]
  ["Deposit $50 into checking"]
────────────────────────
ACTIONS:
  [📊 My Accounts]
  [📋 Recent Transactions]
  [💳 Check Balance]
  ...
────────────────────────
LEARN & EXPLORE:
  [OAuth Login Flow]
  [Token Exchange]
  ...
```

After (no labels, visual grouping only):
```
[🔄 Refresh]  [🔐 Sign in]     ← tiny icon row, no label
────────────────────────
["Show me my accounts"]
["Transfer $100 to savings"]
["Deposit $50 into checking"]
────────────────────────
[📊 My Accounts]
[📋 Recent Transactions]
[💳 Check Balance]
...
────────────────────────
[▾ Learn more]               ← collapsed toggle; expands to show education chips inline
```

**Rules:**
- Remove all `ba-left-label` elements for "Session", "Try asking:", "Actions:"
- Replace "Learn & Explore" section with a `[▾ Learn more]` toggle chip: clicking it inline-expands education command chips without opening a new panel
- SESSION buttons ("Refresh access token" → "Refresh", "Sign in again" → "Sign in") placed in a compact flex row at the top of the rail — no label, only tooltip
- Dividers remain between groups

### Learn More Toggle

- **Collapsed state (default):** Single `[▾ Learn more]` chip in the same style as `ba-action-item`
- **Expanded state:** The chip becomes `[▴ Learn more]`, and education command chips render below it inline — no separate section label
- **State:** tracked with a new `showLearnMore` boolean state (default: `false`)
- Toggle flips between collapsed/expanded on click — does NOT navigate anywhere

### Session Actions Compact Row

```css
.ba-session-row {
  display: flex;
  gap: 4px;
}
.ba-session-row .ba-action-item {
  flex: 1;
  text-align: center;
  padding: 5px 4px;
  font-size: 10px;
}
```

Both SESSION buttons sit in a `div.ba-session-row` — no "Session" label above them.
