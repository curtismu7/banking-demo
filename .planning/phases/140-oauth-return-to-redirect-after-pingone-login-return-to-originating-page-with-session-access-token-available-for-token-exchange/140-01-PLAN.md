<objective>
Strip `?oauth=success` / `?oauth=<any>` from the browser URL after a PingOne login redirect, without disrupting the session-retry logic.

**Root cause audit (Phase 140 research):**
The BFF `oauthUser.js` already fully implements `return_to`:
- Login handler (line ~201): `sanitizePostLoginReturnPath(req.query.return_to)` → stored in `req.session.postLoginReturnToPath`
- Callback (line ~547): `res.redirect(${origin}${postLoginReturnToPath}?oauth=success${ssoParam})`
- `sanitizePostLoginReturnPath` validates same-origin paths, allows `/pingone-test` and `/mfa-test`

The BFF is correct. The **only gap** is the frontend: `?oauth=success` is appended to every post-login URL but never cleaned up, leaving it in the browser URL bar indefinitely.

Compare:
- `sso_silent=1` → stripped by existing one-shot useEffect at line ~378 in App.js ✅
- `oauth=success` → NOT stripped anywhere ❌ (this plan fixes it)

**What does NOT need changing:**
- `banking_api_server/routes/oauthUser.js` — already correct
- PingOneTestPage login button — already uses `?return_to=/pingone-test`
- App.js session-retry logic — `oauthSuccess` is captured inside the effect closure before cleanup runs; safe

Purpose: Clean URL after OAuth return so `/pingone-test` (not `/pingone-test?oauth=success`) is shown to the user.
Output: One-shot useEffect in App.js, 6 lines.

After the sso_silent useEffect (currently ending around line 390 with `}, []`), add a new one-shot useEffect that strips the `oauth` query param from the URL:

```js
/** OAuth success landing: strip ?oauth= param from URL — same pattern as sso_silent handler above. */
useEffect(() => {
  if (typeof window === 'undefined') return;
  const params = new URLSearchParams(window.location.search || '');
  if (!params.has('oauth')) return;
  params.delete('oauth');
  const newSearch = params.toString();
  const newUrl = window.location.pathname + (newSearch ? `?${newSearch}` : '');
  window.history.replaceState(null, '', newUrl);
// eslint-disable-next-line react-hooks/exhaustive-deps -- one-shot on mount only
}, []);
```

Insert it IMMEDIATELY after the sso_silent handler (after its closing `}, []`). Do NOT modify any other code.

Why it's safe:
- The session-retry useEffect (lines ~303–344) captures `oauthSuccess` in its own closure before any of the one-shot effects run (React runs effects in declaration order, all after first paint). By the time this cleanup effect removes `?oauth` from the URL, the session-retry closure has already read it.
- This mirrors exactly the `sso_silent` pattern: params.delete() + window.history.replaceState — no page reload, no state loss.
