    ```markdown
    # BX Finance Postman Collections — Quick-Start Guide

    ## Collections Overview

    | Collection | Audience | Best for |
    |------------|----------|----------|
    | `BX-Finance-1-Exchange-Step-by-Step.postman_collection.json` | Learner | Understanding each OAuth step individually |
    | `BX Finance — 1-Exchange Delegated Chain — pi.flow.postman_collection.json` | Demo runner | Running the full 1-exchange flow quickly |
    | `BX Finance — 2-Exchange Delegated Chain — pi.flow.postman_collection.json` | Demo runner | Running the full 2-exchange (nested delegation) flow |
    | `BX-Finance-Advanced-Utilities.postman_collection.json` | Engineer | PAZ policy decisions, token revocation |

    All collections use: `BX-Finance-Shared.postman_environment.json`

    ---

    ## Prerequisites

    Before running any collection:

    1. **Import the shared environment** — In Postman: Import → select `BX-Finance-Shared.postman_environment.json`. Select it as the active environment in the top-right environment selector.
    2. **Fill in your values** — In the environment editor, set:
       - `PINGONE_ENVIRONMENT_ID` — your PingOne environment ID (Admin → Settings → Environment ID)
       - `PINGONE_CORE_USER_CLIENT_ID` / `PINGONE_CORE_USER_CLIENT_SECRET` — end-user app credentials
       - `PINGONE_CORE_CLIENT_ID` / `PINGONE_CORE_CLIENT_SECRET` — AI agent / BFF app credentials
       - `MCP_CLIENT_ID` / `MCP_CLIENT_SECRET` — MCP service app credentials
       - `TEST_USERNAME` / `TEST_PASSWORD` — test user credentials
       - `PINGONE_CORE_USER_REDIRECT_URI` — must match your PingOne app's allowed redirect URI
       - `ENDUSER_AUDIENCE` — AI Agent resource URI registered in PingOne (e.g., `https://ai-agent.yourdomain.com`)
       - `MCP_RESOURCE_URI` — MCP Server resource URI registered in PingOne
    3. **Set mayAct on your test user** (one-time) — Run "Utility B — Set mayAct on User" from any collection to add the `mayAct` attribute. Required for the token exchange chain to work.

    ---

    ## If you're a Learner (want to understand each OAuth step)

    Use: **`BX-Finance-1-Exchange-Step-by-Step`** + shared env

    Run steps **1a → 1b → 1c → 1d → 2 → 3 → 4** in order.

    Each step saves its output token to a collection variable. After Step 2, inspect the MCP token in Utility A (Introspect) to see the `act` claim showing the delegation chain.

    ---

    ## If you're a Demo Runner (want to run the full flow quickly)

    **1-Exchange flow** — Use: **`BX Finance — 1-Exchange Delegated Chain — pi.flow`** + shared env

    Run steps **1 → 2 → 3 → 4 → 5 → 6 → 7** in order. Steps 1–4 complete the pi.flow authorization; Step 5 performs the RFC 8693 token exchange.

    **2-Exchange flow** — Use: **`BX Finance — 2-Exchange Delegated Chain — pi.flow`** + shared env

    Run steps **1 → 2 → 3 → 4 → 5a → 5b → 6a → 6b → 7 → 8** in order. Steps 5b and 6b are the two RFC 8693 token exchanges that build the nested `act` chain.

    ---

    ## If you're an Engineer (want PAZ or token revocation)

    Use: **`BX-Finance-Advanced-Utilities`** + shared env

    - **PAZ Policy Decision**: Run a 2-exchange pi.flow first to populate `mcp_exchanged_token`. Set `PAZ_DECISION_ENDPOINT_ID` in the environment (find the UUID in PingOne Admin → Authorize → Decision Endpoints). Then run "PAZ Policy Decision" — expected response: 200 with a `decision` field.
    - **Token Revocation**: Set `token_to_revoke` to the token you want to invalidate. Run "Token Revocation" — expected response: 200 empty body.

    ---

    ## Common Errors

    | Error | Cause | Fix |
    |-------|-------|-----|
    | `stale flow_id` / `TIMEOUT` | pi.flow `flow_id` is valid for 120 seconds. Step 1b timed out. | Re-run Step 1a to start a new flow, then immediately run Step 1b. |
    | `Variable not set` (pre-request script throws) | A required collection or environment variable is empty. | Check the error message — it tells you which variable. Open the environment editor and fill in the missing value. |
    | `Environment not selected` | No environment active in Postman — all `{{VARS}}` resolve to empty. | Select `BX Finance — Shared` from the environment dropdown (top-right in Postman). |
    | `invalid_grant` on token exchange | Subject Token expired, or `may_act` not set on user, or `resource` audience mismatch. | Re-run the login steps to get a fresh Subject Token. Verify Utility B (Set mayAct) has been run for the test user. Verify ENDUSER_AUDIENCE matches the resource registered in PingOne. |
    | PAZ returns 404 | `PAZ_DECISION_ENDPOINT_ID` is wrong or empty. | Find the correct UUID in PingOne Admin → Authorize → Decision Endpoints → copy the UUID from the endpoint's URL. |
    | Token Revocation returns 401 | `MCP_CLIENT_ID` or `MCP_CLIENT_SECRET` is wrong. | Check MCP client credentials in the shared environment file. |
    ```
