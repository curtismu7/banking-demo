  <files>docs/BX-Finance-Token-Exchange-Customer.drawio</files>
  <action>
    Create a draw.io XML diagram (`docs/BX-Finance-Token-Exchange-Customer.drawio`) showing the two-hop RFC 8693 token exchange flow as a left-to-right swim-lane sequence diagram.

    **Swim lanes (top to bottom):** User Browser / BFF (Banking Backend) / MCP Server / Resource Server

    **Flow — 1-Exchange path:**
    1. User logs in → PingOne issues Subject Token (`sub=user, aud=bff, scope=banking:agent:invoke`)
    2. BFF exchanges Subject Token for MCP Token (RFC 8693 exchange_type=urn:ietf:params:oauth:token-type:access_token)
       - MCP Token: `sub=user, aud=mcp-server, scope=banking:agent:invoke, act={sub: bff-client-id}`
    3. BFF calls MCP Server with MCP Token

    **Flow — 2-Exchange (Delegated Chain) path:**
    4. MCP Server exchanges MCP Token for Resource Token
       - Resource Token: `sub=user, aud=resource-server, scope=banking:read, act={sub: mcp-server, act={sub: agent1}}`
    5. MCP Server calls Resource Server (Banking API) with Resource Token

    **Design requirements:**
    - Minimum two distinct visual paths (1-Exchange label and 2-Exchange label)
    - Each arrow is labelled with the token name and its key claims (sub, aud, scope, act.sub)
    - Color scheme: Blue for User, Green for BFF/MCP, Orange for Resource Server
    - No logos — text labels only
    - Add "RFC 8693 Token Exchange" title at top of diagram

    Write valid draw.io mxGraph XML. The root element is `<mxGraphModel>`. Use `<mxCell>` for shapes and edges. Use swimlane shape for lanes. Geometry must be numeric. Format the XML as human-readable with proper indentation.

    Minimum size: 1600×900 pixels (set `<mxGraphModel width="1600" height="900" ...>`).
    Use `style="rounded=1;arcSize=10;"` for token boxes.
    Use `style="swimlane;"` for swim lanes.
    Arrows: `style="edgeStyle=orthogonalEdgeStyle;"`.

    Label tokens inline on arrow using `label` attribute.
