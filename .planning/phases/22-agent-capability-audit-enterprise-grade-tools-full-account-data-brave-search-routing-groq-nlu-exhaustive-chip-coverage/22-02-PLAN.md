  <name>Task 1: Create braveSearchService.js BFF wrapper</name>
  <files>banking_api_server/services/braveSearchService.js</files>
  <action>
    Create `banking_api_server/services/braveSearchService.js`. This service:

    1. Reads `process.env.BRAVE_SEARCH_API_KEY` (no configStore — this is an external API key).
    2. Exports `async function search(query, { count = 5 } = {})`.
    3. If `BRAVE_SEARCH_API_KEY` is not set, throws `{ code: 'BRAVE_NOT_CONFIGURED', message: 'Brave Search is not configured. Set BRAVE_SEARCH_API_KEY in your environment.' }`.
    4. Uses `fetch` (Node 18+) to call `https://api.search.brave.com/res/v1/web/search` with:
       - Query params: `q=encodeURIComponent(query)&count=count`
       - Headers: `X-Subscription-Token: BRAVE_SEARCH_API_KEY`, `Accept: application/json`
    5. If HTTP response is not OK, throw `{ code: 'BRAVE_API_ERROR', status: response.status, message: 'Brave Search API error' }`.
    6. Parse JSON response and return formatted results:
       ```js
       return {
         query,
         results: (data.web?.results || []).map(r => ({
           title: r.title || '',
           url: r.url || '',
           description: r.description || ''
         }))
       };
       ```
    7. Export: `module.exports = { search, isConfigured: () => !!process.env.BRAVE_SEARCH_API_KEY }`.

    **Security (OWASP A3):** The API key is ONLY used server-side. Never log the key value. Never return the key to the client. The client only sees search results.
  </action>
  <verify>
    `node -e "const s = require('./banking_api_server/services/braveSearchService.js'); console.log(typeof s.search, typeof s.isConfigured)"`
    Should output: `function function`
  </verify>
  <done>
    braveSearchService.js exports search() and isConfigured() functions. Uses fetch with BRAVE_SEARCH_API_KEY header. Returns structured results array. Throws named error objects when not configured or API fails.
  </done>

    **Part A — bankingAgentNl.js:**

    Read `banking_api_server/routes/bankingAgentNl.js`. Find where it processes the resolved `action` from the NLU call and dispatches to banking operations (e.g., a switch/if-else on `action`). Add a handler for `action === 'web_search'`:

    ```js
    if (action === 'web_search') {
      const braveSearchService = require('../services/braveSearchService');
      const query = intent.query || message || '';
      try {
        const searchResult = await braveSearchService.search(query);
        return res.json({
          ok: true,
          action: 'web_search',
          steps: [{ name: 'brave_search', status: 'success' }],
          result: searchResult,
          nlMeta: { ...nlMeta, searchConfigured: braveSearchService.isConfigured() }
        });
      } catch (err) {
        if (err.code === 'BRAVE_NOT_CONFIGURED') {
          return res.json({
            ok: false,
            action: 'web_search',
            steps: [{ name: 'brave_search', status: 'failed' }],
            error: err.message,
            nlMeta
          });
        }
        throw err;
      }
    }
    ```

    Add this block BEFORE the existing `education` / default handler, so web_search is caught before fallthrough.

    **Part B — groqNlIntent.js:**

    Read `banking_api_server/services/groqNlIntent.js`. Find the JSON schema / system prompt that defines valid banking actions. Add `web_search` as a new valid action with shape `{"kind":"banking","banking":{"action":"web_search","query":"<search terms>"}}`.

    ```js
    }
    ```
    Add this AFTER the banking-specific patterns (accounts, balance, etc.) and BEFORE the default education fallback.
