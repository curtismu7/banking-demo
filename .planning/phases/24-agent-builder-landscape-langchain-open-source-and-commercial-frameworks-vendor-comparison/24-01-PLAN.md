    ### Tab 1 — id: 'langchain', label: 'LangChain'

    Content: Deep-dive on the LangChain ecosystem (this demo uses LangChain via `langchain_agent/`).

    **Opening paragraph:** LangChain is the most widely adopted open-source framework for building LLM-powered applications and agents. It provides composable primitives (chains, agents, tools, memory) that work with any LLM provider. The BX Finance demo's `langchain_agent/` uses LangChain to orchestrate the Banking Agent's tool calls.

    **Core concepts (one FrameworkCard or section each):**

    1. **LCEL — LangChain Expression Language**
       - Pipe syntax for composing chains: `chain = prompt | llm | parser`
       - Lazy evaluation: chains are DAGs, not eager function calls
       - Built-in streaming, batching, async support
       - Replaces the older `LLMChain` / `SequentialChain` API
       - Code example:
         ```python
         from langchain_core.prompts import ChatPromptTemplate
         from langchain_openai import ChatOpenAI

         prompt = ChatPromptTemplate.from_template("Answer: {question}")
         llm = ChatOpenAI(model="gpt-4o")
         chain = prompt | llm          # LCEL pipe
         result = chain.invoke({"question": "What is RFC 8693?"})
         ```

    2. **Agents & Tool Use**
       - Agents decide which tool to call based on the LLM's output
       - Tool: any Python function decorated with `@tool` or wrapped in `Tool()`
       - ReAct pattern: Reason → Act → Observe loop (default for `create_react_agent`)
       - Tool result fed back to LLM until final answer
       - Code example:
         ```python
         from langchain.agents import create_react_agent, AgentExecutor
         from langchain_core.tools import tool

         @tool
         def get_balance(account_id: str) -> str:
             """Get the balance for a given account."""
             return "£4,250.00"

         agent = create_react_agent(llm, [get_balance], prompt)
         executor = AgentExecutor(agent=agent, tools=[get_balance])
         executor.invoke({"input": "What is my balance?"})
         ```

    3. **LangGraph — Stateful Agent Workflows**
       - Graph-based state machine for multi-step, multi-agent workflows
       - Nodes = functions/agents; Edges = conditional transitions
       - Supports cycles (loops), persistence (checkpointers), human-in-the-loop
       - Supersedes older `AgentExecutor` for complex workflows
       - Key concepts: `StateGraph`, `MessagesState`, `interrupt_before`, `MemorySaver`

    4. **LangSmith — Observability & Evaluation**
       - Tracing: every LLM call, tool call, and chain step logged with latency + tokens
       - Evaluation: run datasets through chains, score outputs, regression test
       - Playground: interactive prompt iteration in the UI
       - Requires `LANGCHAIN_API_KEY` and `LANGCHAIN_TRACING_V2=true`

    5. **LangServe — Deployment**
       - Turns any LangChain chain/agent into a FastAPI endpoint with one decorator
       - Auto-generates `/invoke`, `/stream`, `/batch`, `/stream_log` routes
       - OpenAPI docs auto-generated
       - Being superseded by LangGraph Platform for production deployments

    **Version note:** LangChain underwent a major redesign in 2024 — `langchain-core` (stable primitives), `langchain` (agent/chain abstractions), `langchain-community` (integrations), `langchain-openai`/etc. (provider packages). Import paths changed significantly from 0.1.x → 0.3.x.

    ### Tab 2 — id: 'opensource', label: 'Open-Source'

    Content: Open-source agent builder alternatives to LangChain. One FrameworkCard per framework.

    **LlamaIndex** (color: orange)
    - Category: RAG & data-first agents
    - Tagline: "Turn your data into AI-queryable knowledge"
    - Focus: connecting LLMs to data sources (PDFs, databases, APIs, web). Best-in-class for RAG pipelines.
    - Key primitives: `VectorStoreIndex`, `QueryEngine`, `RouterQueryEngine`, `SubQuestionQueryEngine`
    - Agents: `FunctionCallingAgent`, `ReActAgent` — tool use on top of indexed data
    - 2024+ evolution: `llama-index-core` split; LlamaCloud for managed indexes
    - Best for: document Q&A, knowledge bases, enterprise data agents

    **Microsoft AutoGen** (color: blue)
    - Category: Multi-agent conversation framework
    - Tagline: "Orchestrate multiple AI agents in collaborative conversations"
    - Focus: multi-agent systems where agents converse with each other to solve tasks
    - Key primitives: `ConversableAgent`, `AssistantAgent`, `UserProxyAgent`, `GroupChat`
    - Unique: agents can execute code (Python, shell) in sandboxed environments
    - AutoGen 0.4+: redesigned with `AutoGen Core` for event-driven agent networks
    - Best for: data analysis pipelines, code generation, research workflows where multiple agents collaborate

    **CrewAI** (color: purple)
    - Category: Role-based multi-agent orchestration
    - Tagline: "Crews of AI agents with roles, goals, and tasks"
    - Focus: structured teams — define Agents with roles (researcher, writer, analyst), assign Tasks, assemble into a Crew
    - Key primitives: `Agent(role=, goal=, backstory=)`, `Task(description=, agent=)`, `Crew(agents=, tasks=, process=)`
    - Processes: Sequential (linear), Hierarchical (manager delegates), Parallel
    - CrewAI Enterprise: hosted control plane with triggers and monitoring
    - Best for: content pipelines, research-to-report workflows, long-horizon tasks with clear role separation

    **Haystack** (color: teal)
    - Category: NLP pipeline framework (both RAG and agents)
    - Tagline: "Production-ready NLP pipelines since before LLMs existed"
    - Focus: modular pipeline graphs with Components; strong NLP roots (QA, summarization, NER)
    - Key primitives: `Pipeline`, `Component`, `Generator`, `Retriever`, `DocumentStore`
    - Haystack 2.0 (2024): fully redesigned; async-first, typed dataclasses, YAML serializable pipelines
    - Integrations: Weaviate, Qdrant, OpenSearch, Elasticsearch, PgVector
    - Best for: production NLP/RAG pipelines; teams that want declarative, serializable pipeline definitions

    **HuggingFace smolagents** (color: amber)
    - Category: Lightweight agentic library
    - Tagline: "Minimal, transparent agents that run HuggingFace models"
    - Focus: simple agents that write and execute Python code to solve tasks (Code Agent pattern)
    - Key primitives: `CodeAgent`, `ToolCallingAgent`, `Tool`, `HfApiModel`
    - Philosophy: agents write python code snippets as actions rather than structured JSON tool calls — more flexible but requires a sandboxed execution environment
    - Best for: research, learning agent internals, HuggingFace model-centric workflows

    **Phidata / Agno** (color: blue)
    - Category: Full-stack agent framework
    - Tagline: "Agents with memory, knowledge, tools, and reasoning baked in"
    - Focus: production agents with built-in memory (PostgreSQL/SQLite), knowledge bases (PDFs, URLs, tables), structured outputs
    - Phidata rebranded to Agno in early 2025
    - Key primitives: `Agent(model=, tools=, knowledge=, storage=)`, `Team` for multi-agent
    - Best for: full-stack agents that need persistent memory + knowledge base without assembling primitives manually

    ### Tab 3 — id: 'commercial', label: 'Commercial'

    Content: Commercial / cloud-hosted agent builder platforms.

    **AWS Bedrock Agents** (color: #f97316 orange)
    - Category: Managed cloud agent service
    - Tagline: "Build agents without managing infrastructure — AWS-native"
    - Key capabilities: Action Groups (Lambda-backed tool definitions), Knowledge Bases (S3+OpenSearch RAG), Guardrails (content safety), Agent collaboration (multi-agent), Session management
    - Model choice: Claude, Llama, Titan, Mistral via Bedrock
    - Auth: IAM-native — no separate identity layer for AWS-internal agents; cross-account agent invocation via resource-based policies
    - Best for: AWS shops that want agent orchestration with zero infrastructure; compliance-heavy orgs (FedRAMP, HIPAA, SOC 2)

    **Microsoft Copilot Studio** (color: #2563eb blue)
    - Category: Low-code agent builder
    - Tagline: "Build custom copilots — from no-code to pro-code"
    - Key capabilities: Generative AI topics (GPT-4o backbone), Plugin actions (OpenAPI connectors), Power Automate flows, MCP support (2025), publish to Teams/web/phone, autonomous triggers
    - Enterprise features: DLP policies, Azure AD integration, audit logs, usage analytics
    - Semantic Kernel: the pro-code SDK underneath Copilot Studio for custom plugins
    - Best for: Microsoft 365 shops wanting to extend Copilot with company-specific data and workflows

    **Google Vertex AI Agent Builder** (color: #16a34a green)
    - Category: Managed cloud agent service
    - Tagline: "Ground agents in your data with Google Search quality"
    - Key capabilities: Data Stores (website, structured data, unstructured), Grounding (Google Search, own data), Playbooks (declarative agent orchestration), Extensions (tool use via OpenAPI), Agent Engine (managed LangChain/LlamaIndex runtime)
    - Evolved from Dialogflow CX + CCAI in 2024
    - Best for: customer-facing agents that need Google Search grounding; GCP shops; agents on Gemini with long context

    **Salesforce Agentforce** (color: #0ea5e9 sky)
    - Category: Enterprise CRM-native agent platform
    - Tagline: "AI agents that take action in your CRM — no prompt engineering"
    - Key capabilities: Agent topics + instructions (declarative), Standard actions (CRM CRUD, flows, Apex), Atlas reasoning engine, Data Cloud grounding (360° customer data), built-in trust layer (PII masking, audit)
    - Agentforce 2.0 (late 2024): background agents, multi-agent coordination, MuleSoft API action library
    - Best for: sales, service, and marketing orgs already on Salesforce; CRM-centric agent workflows

    **Dify** (color: #7c3aed purple)
    - Category: Open-source + SaaS LLM app platform
    - Tagline: "LLM app platform with visual orchestration — open-source or cloud"
    - Key capabilities: Workflow canvas (drag-drop), Chatbot / agent modes, RAG pipeline (native chunking + retrieval), Model provider hub (100+ models), API / iframe embed, self-hostable (Docker)
    - Popular in Asia-Pacific market; large open-source community
    - Best for: teams wanting a visual workflow builder; quick internal tool deployment; self-hosted LLM apps
