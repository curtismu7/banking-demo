    ### Tab 3 — id: 'howllmswork', label: 'How LLMs Work'

    Opening: LLMs (Large Language Models) are neural networks trained to predict the next token in a sequence. Understanding how they work helps you use them better — and explain them to colleagues, customers, and auditors.

    **Section 1 — Transformers (the architecture)**
    - All modern LLMs are based on the Transformer architecture (Vaswani et al., 2017 "Attention Is All You Need")
    - Key insight: **self-attention** — each token can "attend" to every other token in the context window, capturing long-range dependencies
    - Components: Token embeddings → Positional encoding → N × (Multi-head attention → Feed-forward network) → Output projection → Softmax probability distribution over vocabulary
    - Decoder-only transformers (GPT architecture) generate text autoregressively: each new token is predicted from all previous tokens
    - Simple analogy: think of self-attention as "every word in the sentence votes on how much it should influence every other word's meaning"

    **Section 2 — Training pipeline**

    Use this simple flow diagram as a `<pre>` block:
    ```
    1. Pre-training          2. Supervised Fine-tuning    3. RLHF / RLAIF
    ──────────────           ─────────────────────────    ────────────────
    Raw web text             Curated Q&A pairs            Human raters score
    (trillions of tokens)    Instruction following        model outputs
          │                        │                             │
    Predict next token      Learn to follow prompts      Train reward model
    Cross-entropy loss      (SFT on demonstrations)      PPO / DPO to score
          │                        │                             │
    General knowledge         More helpful               Less harmful,
    learned from data        responses                  more aligned
    ```

    - **Pre-training** (weeks/months, millions of $ in compute): The model sees enormous amounts of text and learns to predict the next word. This is where factual knowledge and language patterns are absorbed.
    - **SFT (Supervised Fine-Tuning)**: The pre-trained model is fine-tuned on curated examples of good assistant behaviour — prompt/response pairs written or verified by humans.
    - **RLHF (Reinforcement Learning from Human Feedback)**: Human raters compare pairs of model responses and indicate which is better. A reward model is trained to predict human preference. The LLM is then updated via PPO (or DPO) to maximise the reward signal.
    - **Constitutional AI (Anthropic)**: Instead of human raters for every preference, a set of principles ("the constitution") guides AI-generated feedback — more scalable.

    **Section 3 — Key concepts**
    - **Context window**: The maximum number of tokens the model can "see" at once. GPT-4o: 128K. Gemini 1.5 Pro: 1M. Everything outside the window is forgotten.
    - **Temperature**: Controls randomness. 0 = deterministic (always picks the highest-probability token). 1 = more creative/random. Above 1 = often incoherent.
    - **Top-p (nucleus sampling)**: Picks from the smallest set of tokens whose cumulative probability ≥ p. `top_p=0.9` means "pick from the 90% probability mass."
    - **Tokens vs words**: ~1 token ≈ ¾ of a word in English. "ChatGPT is great!" ≈ 6 tokens. Pricing is per token (input + output).
    - **Hallucination**: The model generates confident-sounding text that is factually wrong. Root cause: the model predicts plausible-sounding sequences, not retrieving ground truth. Mitigations: RAG (ground with retrieved facts), temperature 0, instruction to say "I don't know."
    - **System prompt / role**: The instruction that defines the model's persona, constraints, and available context. Processed before the user's message.

    **Section 4 — Inference concepts**
    - **KV cache**: Cached key-value pairs from the attention mechanism avoid recomputing attention for tokens already processed. Critical for fast generation.
    - **Quantisation**: Reducing model weights from 32-bit to 8-bit or 4-bit floats. Cuts memory 4-8×; small quality loss. GGUF (llama.cpp) is the dominant format for quantised local inference.
    - **Speculative decoding**: A small "draft" model generates candidate tokens; the large model verifies them in parallel — 2-3× throughput gain.
    - **Batch size**: Process multiple requests simultaneously on the same GPU. High batch size = better hardware utilisation; larger latency per individual request.

    **Section 3 — How to choose**

    Prose guidance (condensed):
    - **Best overall quality (commercial):** GPT-4o / Claude 3.5 Sonnet — neck-and-neck; Claude wins on instruction-following and long docs; GPT-4o wins on multimodal and speed.
    - **Reasoning tasks:** o1/o3 (OpenAI) or DeepSeek R1 (open-source MIT) — extended chain-of-thought; higher latency, worth it for complex analysis.
    - **Largest context window:** Gemini 1.5 Pro (1M tokens) — process entire codebases or multi-hour transcripts.
    - **Best open-source general purpose:** Llama 3.1 70B — community, quality, and 128K context.
    - **Best open-source coding:** Qwen2.5-Coder 72B or DeepSeek V3 — state-of-the-art open-source code models.
    - **Cheapest capable model:** GPT-4o mini, Claude 3.5 Haiku, or Mistral Small 3 — all strong for high-volume pipelines.
    - **Local inference / privacy:** Ollama + Llama 3.1 8B or Mistral 7B — runs on 8-16GB RAM with quantisation.
