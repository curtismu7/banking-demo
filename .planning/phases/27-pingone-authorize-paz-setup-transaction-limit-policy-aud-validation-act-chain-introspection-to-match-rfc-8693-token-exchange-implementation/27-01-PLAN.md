<objective>
Fix the RFC 8693 actor claim extraction in the MCP first-tool gate: add `act.sub` fallback for `actClientId` (RFC 8693 §4 specifies `act.sub` as the canonical actor identifier; `act.client_id` is PingOne-specific). Update code comments in both files to document the claim-to-PAZ-parameter mapping precisely.

Purpose: When PingOne issues tokens using RFC 8693 canonical `act.sub` (e.g., for act chain participants identified by URL rather than PingOne Client ID), the PAZ gate correctly passes the actor identity. Matches the RFC 8693 introspection picture shown in the PingOne Authorize setup — AUD = resource URL, act.sub = MCP server URL, act.act.sub = agent URL.
Output:
- `mcpToolAuthorizationService.js` — actClientId extraction uses `act.client_id || act.sub`
- `pingOneAuthorizeService.js` — JSDoc updated to document act.sub as RFC 8693 standard
</objective>
