  <name>Task 1: Create banking_api_server/routes/vercelConfig.js</name>
  <files>banking_api_server/routes/vercelConfig.js</files>
  <action>
    Create `banking_api_server/routes/vercelConfig.js`.

    **Route structure:**

    ```js
    'use strict';
    const express = require('express');
    const router = express.Router();

    const VERCEL_API = 'https://api.vercel.com';

    function getVercelCredentials() {
      const token = process.env.VERCEL_TOKEN;
      const projectId = process.env.VERCEL_PROJECT_ID;
      return { token, projectId, ready: !!(token && projectId) };
    }

    function isSensitiveType(type) {
      return type === 'secret' || type === 'encrypted';
    }
    ```

    **GET `/` route handler:**
    1. Check `getVercelCredentials().ready` — if not ready, return 503 `{ ok: false, error: 'VERCEL_NOT_CONFIGURED', message: 'VERCEL_TOKEN and VERCEL_PROJECT_ID must be set.' }`
    2. Call Vercel API: `GET ${VERCEL_API}/v9/projects/${projectId}/env` with header `Authorization: Bearer ${token}`.
    3. If Vercel API returns non-OK, return 502 with `{ ok: false, error: 'VERCEL_API_ERROR', status: vercelResponse.status }`
    4. Map response `.envs` to: `{ key, type: env.type, hasValue: !!(env.value), value: isSensitiveType(env.type) ? null : (env.value || null), id: env.id }`
    5. Return `{ ok: true, vars: mappedVars }`.

    **PATCH `/:key` route handler:**
    1. Check credentials ready — same 503 if not.
    2. Validate `req.body.value` is a string — 400 if not.
    3. First call GET to find the env var by key (get its `id` and `type`).
    4. If not found, CREATE via POST (body: `{ key, value, type: 'plain', target: ['production', 'preview'] }`).
    5. If found and `isSensitiveType(env.type)`: return 403 `{ ok: false, error: 'CANNOT_UPDATE_SECRET', message: 'Secrets cannot be updated via this API. Re-enter the value directly in Vercel.' }`.
    6. If found and plain: PATCH `${VERCEL_API}/v9/projects/${projectId}/env/${env.id}` with body `{ value: req.body.value }`.
    7. Return `{ ok: true, key, type: env.type || 'plain' }`.

    **Security:**
    - Never log `VERCEL_TOKEN`.
    - Never include env var values in error responses.
    - Both routes are mounted at `/api/admin/vercel-config` behind admin auth middleware (handled in index.js).

    `module.exports = router;`
