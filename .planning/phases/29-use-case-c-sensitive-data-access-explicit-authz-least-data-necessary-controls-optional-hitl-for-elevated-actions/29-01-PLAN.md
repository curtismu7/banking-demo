**accounts.js provisioning** (lines 74-116 area, `provisionDemoAccounts`):
Replace `accountNumber: \`CHK-${uid.toUpperCase()}\`` with a 12-digit numeric string
generated from the uid:
```js
function generateAccountNumber(prefix, uid) {
  // pad uid digits to 10 chars to make a 12-digit number with 2-digit prefix
  const digits = uid.replace(/[^0-9a-f]/g, '').slice(0, 10);
  const n = parseInt(digits, 16) % 9999999999; // keep within 10 digits
  return String(prefix).padStart(2, '0') + String(n).padStart(10, '0');
}
```
- Checking account: prefix `01`, accountNumberFull = `generateAccountNumber('01', uid)`
- Savings account: prefix `02`, accountNumberFull = `generateAccountNumber('02', uid)`

Store BOTH `accountNumberFull` (raw, e.g. `010123456789`) AND `accountNumber`
(masked display value computed once: `'****' + accountNumberFull.slice(-4)`).

   - `swiftCode`, `iban`, `branchName`, `branchCode`, `openedDate`
   - `accountHolderName` — populate from `req.user.name || req.user.sub || ''`

2. **Never include** in GET /api/accounts/my (sensitive — require consent gate):
   - `accountNumberFull` — do NOT expose
   - `routingNumber` — do NOT expose

The response shape for each account becomes:
```js
{
  id: account.id,
  accountType: account.accountType,
  name: account.name,
  balance: account.balance,
  currency: account.currency,
  status: account.status || 'active',
```
