# Plan 33-01: localStorage Persistence + sub/act.sub Display

## Goal

Persist the token chain `history[]` array across page refreshes via localStorage. Fold in clear-on-logout and `sub`/`act.sub` claim display (D-01 through D-04 from CONTEXT.md).

## Implementation

### 1. `TokenChainContext.js` — localStorage persistence

**Storage key constant:**
```js
const TOKEN_CHAIN_HISTORY_KEY = 'tokenChainHistory';
```

**Hydrate on mount (lazy initializer):**
```js
const [history, setHistory] = useState(() => {
  try {
    const stored = localStorage.getItem(TOKEN_CHAIN_HISTORY_KEY);
    return stored ? JSON.parse(stored) : [];
  } catch {
    return [];
  }
});
```

**Write-through via useEffect (debounced at 300ms):**
```js
useEffect(() => {
  const timer = setTimeout(() => {
    try {
      localStorage.setItem(TOKEN_CHAIN_HISTORY_KEY, JSON.stringify(history));
    } catch (e) {
      console.warn('[TokenChain] localStorage write failed:', e.message);
    }
  }, 300);
  return () => clearTimeout(timer);
}, [history]);
```

**Expose clearHistory (for logout):**
```js
const clearHistory = useCallback(() => {
  setHistory([]);
  setEvents([]);
  try { localStorage.removeItem(TOKEN_CHAIN_HISTORY_KEY); } catch {}
}, []);
```

Add `clearHistory` to the context value and export it.

---

### 2. `App.js` — clear on logout

In the `logout()` function (line ~355), add before `window.location.href`:
```js
localStorage.removeItem('tokenChainHistory');
```

This is a belt-and-suspenders clear alongside `clearHistory` from context (which may not be called if the component unmounts before logout completes).

### 3. `TokenChainDisplay.js` — sub / act.sub claim labels

In the claim row renderer (wherever `key` labels are displayed), add human-readable labels:

```js
function claimLabel(key) {
  if (key === 'sub') return 'User ID (sub)';
  if (key === 'act') return 'Delegation (act)';
  // ... existing labels
  return key;
}
```

For `act.sub` specifically — when rendering the `act` claim object, surface the nested `sub` as "Agent ID":
```js

  // Show "Agent ID: <value.sub>" prominently before the full JSON
```

Find the existing act/may_act rendering block in `TokenChainDisplay.js` (around lines 36-105) and add the Agent ID label there.
