<objective>
Add read-only tool access tiers to the MCP server. External agents browsing the `/.well-known/mcp-server` manifest or calling `tools/list` should be able to distinguish which tools are safe to call without sensitive account access vs. which require full user auth and write scopes.

- **MCP-PUB-01**: Add `readOnly: boolean` to `BankingToolDefinition` interface. Tag all tools: `get_my_accounts`, `get_account_balance`, `get_my_transactions` = `readOnly: true`; `create_deposit`, `create_withdrawal`, `create_transfer`, `query_user_by_email` = `readOnly: false`; `get_sensitive_account_details` = `readOnly: false` (contains PII); `sequential_think` = `readOnly: true` (stateless reasoning).
- **MCP-PUB-02**: Thread `readOnly` through `tools/list` response so AI clients see it per tool.
- **MCP-PUB-03**: Update `/.well-known/mcp-server` manifest to group tools into `readOnlyTools[]` and `authenticatedTools[]` with a `publicAccess` summary block.

Purpose: External AI agents (Claude Desktop, Cursor, Windsurf) can introspect which tools require OAuth before attempting them, enabling smarter fallback behavior and safer demo access.
Output: TypeScript tool metadata + manifest v2.
</objective>

async handleListTools(message: ListToolsMessage, _context: MessageHandlerContext): Promise<ListToolsResponse> {
  const bankingTools = this.toolProvider.getAvailableTools();
  const mcpTools: ToolDefinition[] = bankingTools.map(tool => ({
    name: tool.name,
    description: tool.description,
    inputSchema: tool.inputSchema,
    requiresUserAuth: tool.requiresUserAuth,
    requiredScopes: tool.requiredScopes
  }));
  return { id: message.id ?? 'unknown', result: { tools: mcpTools, nextCursor: undefined } };
}
