    ### Server Discovery

    Before connecting your AI client manually, you can inspect the server's published tool surface via the
    well-known discovery endpoint:

    ```bash
    curl http://localhost:8080/.well-known/mcp-server
    ```

    The response describes available tools and their access tiers:

    ```json
    {
      "name": "bx-finance-banking",
      "version": "1.0.0",
      "tools": [
        { "name": "get_my_accounts", "readOnly": true, "requiresUserAuth": true, "requiredScopes": ["banking:accounts:read"] },
        { "name": "sequential_think", "readOnly": true, "requiresUserAuth": false, "requiredScopes": [] }
      ],
      "publicAccess": {
        "readOnlyTools": ["get_my_accounts", "get_account_balance", "get_my_transactions", "sequential_think"]
      },
      "restrictedAccess": {
        "authenticatedTools": ["get_sensitive_account_details", "create_deposit", "create_withdrawal", "create_transfer", "query_user_by_email"]
      },
      "auth": {
        "type": "oauth2",
        "authorizationUrl": "<PINGONE_AUTH_URL>",
        "tokenUrl": "<PINGONE_TOKEN_URL>"
      }
    }
    ```

    `publicAccess.readOnlyTools` are tools that an external agent can list and describe to a user without requiring write permissions. `restrictedAccess.authenticatedTools` require explicit OAuth scopes.

    For the production deployment, replace `http://localhost:8080` with the Render URL (see Deployment section).

    > **MCP Spec reference:** [Model Context Protocol — Server discovery](https://modelcontextprotocol.io/specification#server-discovery)
    ```
