<objective>
Create a comprehensive C4 architecture diagram (draw.io format) for the BX Finance Banking Demo, covering all four C4 levels. Build an education panel that displays the diagram with interactive zoom and layer navigation.

Purpose: Architects and developers visiting the demo need a visual top-down understanding of how the React SPA, Express BFF, MCP Server, PingOne, and LLM provider fit together. The C4 model provides a standardized way to communicate this at multiple zoom levels.

Output: draw.io C4 diagram file, exported SVG, and ArchitectureDiagramPanel.js education component.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Create C4 draw.io diagram</name>
  <files>docs/diagrams/c4-banking-demo.drawio</files>
  <action>
Create the draw.io XML file with four tabs/pages for the four C4 levels.

**Level 1 — System Context:**
- Person: Banking User (customer or admin)
- System: BX Finance Banking Demo
- External System: PingOne (SSO, MFA, Management API)
- External System: LLM Provider (OpenAI/Anthropic)
- Relationships: User → Banking Demo (uses), Banking Demo → PingOne (authenticates via), Banking Demo → LLM (AI agent queries)

**Level 2 — Container:**
- banking_api_ui (React SPA) — serves UI, session cookies only
- banking_api_server (Express BFF) — OAuth, tokens, API proxy, agent orchestration
- banking_mcp_server (TypeScript MCP) — WebSocket, tool registry, auth challenges
- PingOne Authorization Server — OAuth 2.0 / OIDC
- PingOne Management API — user CRUD, MFA
- LLM API — chat completions
- Vercel (hosting) — serverless functions + static assets
- Upstash Redis — session store

**Level 3 — Component (BFF internals):**
- OAuth routes (login, callback, logout, refresh)
- Token exchange service (1-exchange, 2-exchange)
- CIBA service (backchannel auth)
- MFA service (deviceAuthentications API)
- Agent service (LangChain, tool calling)
- ConfigStore (SQLite/KV persistence)
- DataStore (accounts, transactions)
- Session middleware (Upstash Redis)

**Level 4 — Code (key services):**
- oauthService.js — PKCE, token management
- agentMcpTokenService.js — token exchange orchestration
- mfaService.js — PingOne MFA flows
- tokenChainService.js — token event tracking
- bankingAgentService.js — LangChain agent
- pingOneClientService.js — Management API client

Use C4 PlantUML-style shapes and colors (blue for internal, grey for external).
  </action>
