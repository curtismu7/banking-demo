## Demo Data Persistence

The banking API server now supports persistent demo accounts that survive server restarts:

### Local Development (SQLite)

When running locally, demo accounts are stored in a SQLite database at `data/persistent/demoAccounts.db`. The database is created automatically on first startup.

### Vercel Deployment (Environment Variables)

When deployed to Vercel, demo accounts are stored in the `DEMO_ACCOUNTS` environment variable as a JSON string.

#### Setting up DEMO_ACCOUNTS on Vercel

1. Go to your Vercel project dashboard
2. Navigate to Settings → Environment Variables
3. Add a new variable:
   - **Name**: `DEMO_ACCOUNTS`
   - **Value**: JSON array of account objects

#### Example DEMO_ACCOUNTS JSON

```json
[
  {
    "id": "acc_001",
    "userId": "user_001",
    "accountType": "checking",
    "accountNumber": "1234567890123456",
    "routingNumber": "021000021",
    "balance": 2500.00,
    "currency": "USD",
    "status": "active",
    "createdAt": "2024-01-15T10:30:00Z"
  },
  {
    "id": "acc_002",
    "userId": "user_001",
    "accountType": "savings",
    "accountNumber": "9876543210987654",
    "routingNumber": "021000021",
    "balance": 15000.00,
    "currency": "USD",
    "status": "active",
    "createdAt": "2024-01-15T10:30:00Z"
  }
]
```

### Migration

On server startup, the system automatically migrates existing in-memory demo accounts to the appropriate persistent storage:
- Local: Migrates to SQLite database
- Vercel: Logs instructions for manual environment variable setup

### API Endpoints

The demo accounts API endpoints work with both storage backends:

- `GET /api/demo/accounts` - List accounts (includes backend info)
- `POST /api/demo/account` - Create new account
- `DELETE /api/demo/account/:id` - Delete account

### UI Management

The Demo Data page in the banking UI shows:
- Current storage backend (SQLite or Vercel)
- Account count and last migration time
- Export to Vercel button (when using SQLite locally)
- Import from Vercel button (when using Vercel)
```
