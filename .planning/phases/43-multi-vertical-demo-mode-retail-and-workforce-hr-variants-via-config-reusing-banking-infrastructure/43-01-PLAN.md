<objective>
Create a vertical configuration system that lets the demo switch between Banking, Retail, and Workforce (HR) modes via config — reusing the same infrastructure, OAuth flows, and MCP server, but swapping terminology, account types, transaction labels, theme colors, and agent persona.

Purpose: Demonstrate that the PingOne + MCP architecture is not banking-specific. A single config switch shows the same identity and AI agent patterns applied to retail or workforce use cases, making the demo relevant to any audience vertical.

Output: verticalConfigService.js, vertical JSON configs, REST API endpoints, and server.js mount.
</objective>

- `getActiveVertical()` — returns current vertical ID from configStore
- `setActiveVertical(verticalId)` — validates and sets in configStore
- `getVerticalConfig(verticalId?)` — returns full config for active or specified vertical
- `mapTerm(term)` — maps generic term to vertical-specific terminology
