<objective>
Implement RFC 9728 Protected Resource Metadata endpoint on the BFF. This is the machine-readable document that lets OAuth clients (agents, MCP servers) discover how to access this resource server — which AS to use, which scopes are available, and what bearer methods are accepted.

Purpose: RFC 9728 is required for standards-compliant MCP server integration. Without it, MCP clients cannot auto-discover auth requirements.

Output: GET /.well-known/oauth-protected-resource returning valid RFC 9728 JSON, plus /api/rfc9728 proxy for SPA use.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Create routes/rfc9728.js</name>
  <files>banking_api_server/routes/rfc9728.js</files>
  <action>
Create Express router that builds and serves RFC 9728 metadata:

```javascript
const express = require('express');
const router = express.Router();
const configStore = require('../services/configStore');

router.get('/', (req, res) => {
  const envId = configStore.getEffective('pingone_environment_id') || process.env.PINGONE_ENVIRONMENT_ID;
  const region = configStore.getEffective('pingone_region') || process.env.PINGONE_REGION || 'com';
  const resourceUrl = process.env.PUBLIC_APP_URL || `${req.protocol}://${req.get('host')}`;
  const asUrl = `https://auth.pingone.${region}/${envId}/as`;

  const metadata = {
    resource: resourceUrl,
    authorization_servers: [asUrl],
    bearer_methods_supported: ['header'],
    scopes_supported: [
      'openid', 'profile', 'email',
      'banking:read', 'banking:write', 'banking:admin',
      'banking:sensitive', 'ai_agent'
    ],
    resource_documentation: `${resourceUrl}/docs`,
  };

  res.set({
    'Content-Type': 'application/json; charset=UTF-8',
    'Cache-Control': 'public, max-age=3600',
    'Access-Control-Allow-Origin': '*',
  });
  res.json(metadata);
});

module.exports = router;
```
