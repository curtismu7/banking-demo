<objective>
Standardize the names used for all PingOne applications, resource servers, and scopes across the provisioning service, documentation, and code constants. Inconsistent naming causes confusion when auditing PingOne environments or comparing with documentation.

Purpose: Anyone looking at the PingOne console should immediately recognize "BX Finance Admin App" as the admin OIDC client without guessing.

Output: CANONICAL_APP_NAMES constant in config/scopes.js, pingoneProvisionService.js updated to use it, and docs/PINGONE_APP_CONFIG.md with the naming table.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Add CANONICAL_APP_NAMES to config/scopes.js</name>
  <files>banking_api_server/config/scopes.js</files>
  <action>
Add at the end of config/scopes.js (before module.exports):

```javascript
// Canonical PingOne resource names — used by provisioning service and documentation
const CANONICAL_APP_NAMES = {
  ADMIN_APP:    'BX Finance Admin App',
  USER_APP:     'BX Finance User App',
  AGENT_APP:    'BX Finance Agent App',
  WORKER_APP:   'BX Finance Worker App',
};

const CANONICAL_RESOURCE_NAMES = {
  BANKING_API:  'BX Finance Banking API',
  AUDIENCE:     'banking_api_enduser',
};

const CANONICAL_SCOPES = [
  'banking:read',
  'banking:write',
  'banking:admin',
  'banking:sensitive',
  'ai_agent',
];
```

Export these in module.exports alongside existing exports.
  </action>
  <done>CANONICAL_APP_NAMES, CANONICAL_RESOURCE_NAMES, and CANONICAL_SCOPES added to config/scopes.js.</done>

Add or update a "Canonical Resource Naming" section in docs/PINGONE_APP_CONFIG.md:

```markdown
## Canonical Resource Naming

All PingOne resources created by the setup wizard or manually follow this naming convention:

| Resource Type | Canonical Name | Notes |
|---|---|---|
| Admin OIDC App | BX Finance Admin App | WEB_APP, PKCE, admin scopes |
| User OIDC App | BX Finance User App | WEB_APP, PKCE, user scopes |
| Agent OIDC App | BX Finance Agent App | WEB_APP, PKCE, ai_agent scope |
| Worker App | BX Finance Worker App | WORKER type, client credentials |
| Resource Server | BX Finance Banking API | audience: banking_api_enduser |

### Scope Naming

| Scope | Purpose |
|---|---|
| banking:read | Read accounts and transactions |
| banking:write | Create transfers, update accounts |
| banking:admin | Admin-only operations |
| banking:sensitive | Access masked sensitive fields |
| ai_agent | AI agent identity marker |
```
