<objective>
Build the BFF provisioning service and SSE streaming API for the setup wizard. The service creates all required PingOne resources (Admin app, User app, Resource Server, scopes, demo users) via Management API, writes .env or sets Vercel env vars, and streams live progress back to the UI.

Purpose: New deployers currently must manually create PingOne apps, resource servers, and scopes in the PingOne Console — a tedious, error-prone process. This wizard automates the entire setup from a single "Run Setup" button.

Output: pingoneProvisionService.js with full PingOne provisioning, setupWizard.js with SSE streaming, server.js mount.
</objective>

Create provisioning service that performs all PingOne setup via Management API.

**Worker token acquisition:**
```js
async function getWorkerToken(envId, clientId, clientSecret, region) {
  const res = await axios.post(
    `https://auth.pingone.${region}/${envId}/as/token`,
    'grant_type=client_credentials',
    {
      auth: { username: clientId, password: clientSecret },
      headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
      timeout: 10000
    }
  );
  return res.data.access_token;
}
```

**provisionEnvironment(config, onStep):**
Main orchestrator. `config` contains worker credentials, public URL, Vercel token, etc. `onStep` is a callback for each progress event.

Steps (each emits SSE event):
1. Validate worker credentials (probe Management API)
2. Get default population ID
3. Create Resource Server "Super Banking API" (or skip if exists)
4. Create banking scopes on resource server
5. Create Admin OIDC app "Super Banking Admin App" (or skip)
6. Set Admin app redirect URIs + PKCE config
7. Grant scopes to Admin app
8. Create User OIDC app "Super Banking User App" (or skip)
9. Set User app redirect URIs + PKCE config
10. Grant scopes to User app
11. Create demo user "bankuser" (or skip)
12. Set bankuser password
13. Create demo user "bankadmin" (or skip)
14. Set bankadmin password
15. Write .env file (local) or set Vercel env vars (Vercel)
16. Done — return provisioned IDs

**PingOne API patterns:**
- Create app: `POST /v1/environments/{envId}/applications`
- Update app: `PUT /v1/environments/{envId}/applications/{appId}` (PUT not PATCH!)
- Create resource: `POST /v1/environments/{envId}/resources`
- Create scope: `POST /v1/environments/{envId}/resources/{resId}/scopes`
- Grant scopes: `POST /v1/environments/{envId}/applications/{appId}/grants`
- Create user: `POST /v1/environments/{envId}/users`
- Set password: `PUT /v1/environments/{envId}/users/{userId}/password` with `Content-Type: application/vnd.pingidentity.password.set+json`
- List resources: `GET /v1/environments/{envId}/resources` (check exists)
- List apps: `GET /v1/environments/{envId}/applications` (check exists)

**Idempotency:**
Before each create, check if resource exists by name. If exists, skip with ⚠️ status and include resourceKey for Recreate option.

**recreateResource(config, resourceKey):**
Deletes the specified resource and re-creates it. Used by the "Recreate" button in the UI.

**Error handling:**
Each step wrapped in try/catch. On error: emit ❌ event with error message, continue to next step if non-fatal.

**POST /api/admin/setup/run:**
```js
router.post('/run', authenticateToken, async (req, res) => {
  const { envId, workerClientId, workerClientSecret, region, publicAppUrl, vercelToken, vercelProjectId } = req.body;
  
  // Validate required fields
  if (!envId || !workerClientId || !workerClientSecret) {
    return res.status(400).json({ error: 'Missing required credentials' });
  }
  
  // SSE headers
  res.writeHead(200, {
    'Content-Type': 'text/event-stream',
    'Cache-Control': 'no-cache',
    'Connection': 'keep-alive',
    'X-Accel-Buffering': 'no'
  });
  
  const onStep = (step) => {
    res.write(`data: ${JSON.stringify(step)}\n\n`);
  };
  
  try {
    const result = await provisionEnvironment({
      envId, workerClientId, workerClientSecret,
      region: region || 'com',
      publicAppUrl: publicAppUrl || process.env.PUBLIC_APP_URL || 'http://localhost:3000',
      vercelToken, vercelProjectId,
      isVercel: !!process.env.VERCEL
    }, onStep);
    
    onStep({ step: 'complete', icon: '✅', message: 'Setup complete!', result });
  } catch (err) {
    onStep({ step: 'error', icon: '❌', message: `Setup failed: ${err.message}` });
  }
  
  res.write('data: [DONE]\n\n');
  res.end();
});
```
