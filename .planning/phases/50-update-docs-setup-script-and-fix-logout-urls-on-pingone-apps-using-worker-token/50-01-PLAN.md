Fix logout URLs on PingOne apps using the worker token (Management API), update all setup documentation to reflect the current state of the demo, and ensure the README quick-start is accurate.

Purpose: Logout currently fails silently because PingOne apps don't have postLogoutRedirectUris configured. This phase fixes that programmatically and ensures all documentation is up to date so new deployers have a smooth experience.

Output: pingoneAppConfigService.js for app config management, docs/SETUP.md comprehensive guide, docs/PINGONE_APP_CONFIG.md reference, README updates.
</objective>

<task type="auto">
  <name>Task 1: Create pingoneAppConfigService.js</name>
  <files>banking_api_server/services/pingoneAppConfigService.js</files>
  <action>
Create service for reading and updating PingOne app configurations.

**getAppConfig(appId):**
- GET /v1/environments/{envId}/applications/{appId}
- Returns full app configuration including redirect URIs, logout URLs, grant types

**fixLogoutUrls(appId, publicAppUrl):**
- Reads current app config
- Calculates correct logout URLs:
  - postLogoutRedirectUris: [publicAppUrl, `${publicAppUrl}/login`, 'http://localhost:3000', 'http://localhost:3001']
  - signOffUrls: same as postLogoutRedirectUris
- PUT /v1/environments/{envId}/applications/{appId} with updated config
- Returns { before, after, changed: boolean }

**auditAppConfig(appId):**
- Reads app config and checks for common issues:
  - Missing logout URLs
  - Missing redirect URIs for localhost
  - PKCE not enabled
  - Wrong grant types
  - Missing required scopes
- Returns structured audit report

**updateAppConfig(appId, updates):**
- PUT /v1/environments/{envId}/applications/{appId}
- Merges updates into existing config
- Returns updated config

Uses getManagementToken() from pingOneClientService for auth.
  </action>
  <done>pingoneAppConfigService.js created.</done>

<task type="auto">
  <name>Task 2: Create appConfig.js route</name>
  <files>banking_api_server/routes/appConfig.js</files>
  <action>
Create admin API routes:

- `GET /api/admin/app-config/:appType` — get config for 'admin' or 'user' app
  - Resolves appId from configStore (admin_client_id or user_client_id)
  - Returns PingOne app configuration

- `POST /api/admin/fix-logout-urls` — fix logout URLs on both apps
  - Reads publicAppUrl from configStore or request body
  - Calls fixLogoutUrls for both admin and user apps
  - Returns report of changes

- `GET /api/admin/app-config/audit` — audit both apps for issues
  - Returns structured audit report

All routes protected by authenticateToken + admin check.
Mount in server.js at `/api/admin/app-config`.
