<objective>
Two additions to Phase 52:

1. **MCP tools load gate** — `GET /api/mcp/inspector/tools` requires MFA verification when
   step-up is enabled. Returns `{ tools: [], mfa_required: true }` if not verified.
   BankingAgent.js handles the `mfa_required` response by triggering the step-up flow and
   retrying the tools fetch after approval (same `cibaStepUpApproved` pattern as write ops).

2. **stepUpMethod config in SecuritySettings** — Admins can choose the MFA method
   (`email` | `ciba` | `pingone-mfa`) from the Security Settings admin page.
   The `pingone-mfa` option unlocks the full PingOne deviceAuthentications flow built in 52-01/52-03.

Purpose: Round out the MFA demo by gating even tool discovery behind MFA, and give admins
a UI toggle to switch between legacy email OTP, CIBA, and the new PingOne MFA approach.

</objective>
