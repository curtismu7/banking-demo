## Objective

Simplify scope narrowing logic and implement explicit scope-audience mapping per RFC 8707 resource indicators, removing multi-layer fallbacks and hard-coded defaults while maintaining RFC 8693 compliance.

**Purpose**: AUDIT-03 requirement — "Scope Narrowing and Audience Handling Analysis"

**Output**: Simplified scope logic, explicit mapping, validation functions, documentation

**Goal**: Replace implicit fallbacks with explicit configuration and validation

```javascript
/**
 * RFC 8707: Explicit scope-audience mapping
 * Maps each valid scope to the audiences where it can be used
 */
const ALLOWED_SCOPES_BY_AUDIENCE = {
  // Agent Gateway (Step 1 actor token)
  'https://agent-gateway.example.com': [
    'banking:agent:invoke',
    'ai_agent',
  ],

  // AI Agent Intermediate (Step 2 exchange output)
  'https://ai-agent-gateway.example.com': [
    'banking:read',
    'banking:write',
    'banking:agent:invoke',
  ],

  // MCP Gateway (Step 3 actor token)
  'https://mcp-gateway.example.com': [
    'banking:mcp:invoke',
    'mcp_resource_access',
  ],

  // MCP Resource Server (Step 4 final)
  'https://resource.example.com/mcp': [
    'get_accounts:read',
    'transfer:execute',
    'check:read',
  ],
};

/**
 * No hard-coded default scopes
 * All scopes must be: (1) granted by user, (2) applicable to audience
 */
const validateScopeAudience = (scopes, audience) => {
  if (!scopes || scopes.length === 0) {
    throw new Error(\`SCOPE_ERROR: No scopes provided for audience \${audience}\`);
  }

  const allowedForAudience = ALLOWED_SCOPES_BY_AUDIENCE[audience];
  if (!allowedForAudience) {
    throw new Error(\`AUDIENCE_ERROR: Unknown audience \${audience}\`);
  }

  const allowedSet = new Set(allowedForAudience);
  const validScopes = scopes.filter(s => allowedSet.has(s));

  if (validScopes.length === 0) {
    throw new Error(
      \`SCOPE_MISMATCH: User scopes [\${scopes.join(', ')}] \\
       do not match allowed scopes for \${audience} [\${allowedForAudience.join(', ')}]\`
    );
  }

  return {
    valid: true,
    scopes: validScopes,
    narrowed: validScopes.length < scopes.length,
  };
};

module.exports = {
  // ... existing exports
  ALLOWED_SCOPES_BY_AUDIENCE,
  validateScopeAudience,
};
```

```javascript
describe('Scope & Audience Mapping (RFC 8707)', () => {
  const testAudience = 'https://ai-agent-gateway.example.com';
  
  it('should validate scopes match audience', () => {
    const result = configStore.validateScopeAudience(
      ['banking:read', 'banking:write'],
      testAudience
    );
    expect(result.valid).toBe(true);
    expect(result.scopes).toEqual(['banking:read', 'banking:write']);
    expect(result.narrowed).toBe(false);
  });

  it('should narrow scopes to audience allowlist', () => {
    const result = configStore.validateScopeAudience(
      ['banking:read', 'banking:agent:invoke', 'invalid:scope'],
      testAudience
    );
    expect(result.valid).toBe(true);
    expect(result.scopes).toContain('banking:read');
    expect(result.scopes).not.toContain('invalid:scope');
    expect(result.narrowed).toBe(true);
  });

  it('should reject scopes with no audience match', () => {
    expect(() => {
      configStore.validateScopeAudience(
        ['invalid:scope', 'another:invalid'],
        testAudience
      );
    }).toThrow(/SCOPE_MISMATCH/);
  });

  it('should reject unknown audiences', () => {
    expect(() => {
      configStore.validateScopeAudience(
        ['banking:read'],
        'https://unknown.example.com'
      );
    }).toThrow(/AUDIENCE_ERROR/);
  });

  it('should reject empty scope list', () => {
    expect(() => {
      configStore.validateScopeAudience([], testAudience);
    }).toThrow(/No scopes provided/);
  });
});
```

```markdown
# Scope-Audience Mapping Reference (RFC 8707)

## Overview

This document maps each OAuth scope to the audiences where it can be applied per RFC 8707 "OAuth 2.0 Resource Indicators" and RFC 8693 token exchange.

## Scope-Audience Mapping Table

### Agent Gateway (Step 1)
Audience: \`https://agent-gateway.example.com\`

| Scope | Purpose | Valid |
|-------|---------|-------|
| banking:agent:invoke | AI Agent authorization | ✅ Yes |
| ai_agent | Legacy AI Agent scope | ✅ Yes |
| banking:read | User data access | ❌ No |
| transfer:execute | Transfer permission | ❌ No |

### AI Agent Intermediate (Step 2)
Audience: \`https://ai-agent-gateway.example.com\`

| Scope | Purpose | Valid |
|-------|---------|-------|
| banking:read | User account/transaction reading | ✅ Yes |
| banking:write | User data modification | ✅ Yes |
| banking:agent:invoke | Delegation scope | ✅ Yes |
| transfer:execute | Transfer permission | ❌ No |

### MCP Gateway (Step 3)
Audience: \`https://mcp-gateway.example.com\`

| Scope | Purpose | Valid |
|-------|---------|-------|
| banking:mcp:invoke | MCP authorization | ✅ Yes |
| mcp_resource_access | Resource access token | ✅ Yes |
| banking:read | User data access | ❌ No |

### MCP Resource Server (Step 4)
Audience: \`https://resource.example.com/mcp\`

| Scope | Purpose | Valid |
|-------|---------|-------|
| get_accounts:read | List user accounts | ✅ Yes |
| transfer:execute | Execute transfers | ✅ Yes |
| check:read | Check reading | ✅ Yes |
| banking:read | Generic banking | ❌ No |

## Validation Rules

1. **No Scope Escalation**: Scopes can only be maintained or narrowed, never escalated
2. **Audience-based Filtering**: Only scopes valid for the target audience are included
3. **Explicit Allow List**: Only scopes in the allowlist for an audience are permitted
4. **No Hard-Coded Defaults**: Missing scopes result in error, not silent fallback
5. **Narrowing is Logged**: Events logged when scopes are narrowed

- ALLOWED_SCOPES_BY_AUDIENCE — Mapping table
- validateScopeAudience() — Validation function
```

### Trust Boundaries

| Boundary | Protection |
|----------|-----------|
| User → Scope List | OAuth 2.0 authorization grant |
| Scope → Audience | Explicit validation mapping |
| Audience → Resource Server | JWT aud claim matching |

### STRIDE Register

| Threat | Type | Mitigation |
|--------|------|-----------|
| Scope escalation | Elevation | RFC 8693 narrowing enforced by explicit mapping |
| Unknown audience | Spoofing | Reject unknown audiences, explicit allowlist only |
| Hard-coded defaults | Tampering | Removed; validation fails if scopes don't match |
