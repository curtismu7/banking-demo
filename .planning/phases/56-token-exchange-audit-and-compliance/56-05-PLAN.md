/**
 * Get error details and remediation steps for an error code
 */
const getErrorDetails = (errorCode) => {
  const config = ERROR_CODES[errorCode];
  if (!config) {
    return {
      http_status: 500,
      oauth_error: 'server_error',
      description: 'Unknown error code',
      category: 'Server',
    };
  }
  return config;
};

/**
 * Map internal error categories to standardized codes
 */
const mapErrorToCode = (errorMessage, context = {}) => {
  // Configuration errors
  if (errorMessage.includes('credentials not configured')) return 'config.missing_credentials';
  if (errorMessage.includes('Invalid audience')) return 'config.invalid_audience';

  // RFC 8693 errors
  if (errorMessage.includes('invalid_client')) return 'invalid_client';
  if (errorMessage.includes('invalid_grant')) return 'invalid_grant';
  if (errorMessage.includes('invalid_scope') || errorMessage.includes('SCOPE_MISMATCH')) return 'invalid_scope';
  if (errorMessage.includes('May_act')) return 'invalid_grant';
  if (errorMessage.includes('Audience mismatch')) return 'invalid_scope';
  if (errorMessage.includes('Token expired')) return 'invalid_token';
  if (errorMessage.includes('access_denied')) return 'access_denied';

  // Default to server error
  return 'server_error';
};

module.exports = {
  // ... existing exports
  ERROR_CODES,
  getErrorDetails,
  mapErrorToCode,
};

# Error Codes and Remediation Guide (RFC 8693 §5.2)

## Overview

This document documents all OAuth token exchange error codes returned by the BX Finance Banking Demo, along with common causes and remediation steps.

All error codes comply with RFC 8693 §5.2 "Error Response" unless marked as "Custom" extensions.

## Error Code Categorization

| Category | Description | HTTP Status |
|----------|-------------|-------------|
| **Configuration** | Application setup issues | 500 |
| **Authentication** | Client or token auth failed | 401 |
| **Authorization** | User/client not authorized | 403 |
| **Scope** | Scope mismatch or insufficient | 400 |
| **Request** | Malformed or unsupported request | 400 |
| **Server** | Internal server error | 500 |

## Error Reference Table

### Configuration Errors

| Error Code | HTTP | OAuth Error | Description | Occurs When |
|------------|------|-------------|-------------|-------------|
| config.missing_credentials | 500 | server_error | Application credentials not configured | PINGONE_AI_AGENT_CLIENT_ID or AGENT_OAUTH_CLIENT_ID not set |
| config.invalid_audience | 500 | server_error | Invalid or missing token audience | PINGONE_AGENT_GATEWAY_AUDIENCE or other audience env vars not set |

**Remediation for config.missing_credentials**:
1. Check environment variables are set: \`echo \$PINGONE_AI_AGENT_CLIENT_ID\`
2. Verify PingOne Admin → Applications → Super Banking AI Agent exists
3. Copy Client ID and Secret to environment
4. Restart server: \`npm run start\`

**Remediation for config.invalid_audience**:
1. Check PingOne Admin → Applications → Super Banking AI Agent
2. Go to Resource Indicators tab
3. Verify PINGONE_AGENT_GATEWAY_AUDIENCE matches one of the URIs
4. Update environment variable if mismatch
5. Restart server

### Authentication Errors

| Error Code | HTTP | OAuth Error | Description | Occurs When |
|------------|------|-------------|-------------|-------------|
| invalid_client | 401 | invalid_client | Client authentication failed | Client ID/Secret invalid or app not registered in PingOne |
| invalid_token | 401 | invalid_token | Token is expired or revoked | User's original token is invalid or expired |

**Remediation for invalid_client**:
1. Verify Client ID is correct: Admin UI Configuration page
2. Verify Client Secret is current (may need to rotate in PingOne)
3. Check app registration in PingOne Admin → Applications
4. Ensure app has Grant type: Client Credentials
5. Restart server and retry

**Remediation for invalid_token**:
1. User: Log in again to get fresh token
2. Check user record exists in PingOne
3. Verify user has not been disabled
4. Check token expiration (typically 1 hour)

### Authorization Errors

| Error Code | HTTP | OAuth Error | Description | Occurs When |
|------------|------|-------------|-------------|-------------|
| unauthorized_client | 403 | unauthorized_client | Client not authorized for this flow | Client app not granted token_exchange grant type |
| access_denied | 403 | access_denied | Resource owner denied the request | User denied consent or may_act not set |

**Remediation for unauthorized_client**:
1. Check PingOne Admin → Applications → [app name] → Grant Types
2. Verify "Token Exchange" grant type is enabled
3. If not, enable and save
4. Restart server

**Remediation for access_denied**:
1. User: Go to Admin UI → DemoData page
2. Toggle "2-Exchange Mode" ON
3. Wait 30 seconds for token refresh
4. Try agent operation again

### Scope Errors

| Error Code | HTTP | OAuth Error | Description | Occurs When |
|------------|------|-------------|-------------|-------------|
| invalid_scope | 400 | invalid_scope | Requested scope invalid or exceeds grant | Scope doesn't match audience or not in user token |
| insufficient_scope | 403 | insufficient_scope | Token lacks required scope | User token doesn't have scope needed for resource |

**Remediation for invalid_scope**:
1. Check scope requested matches ALLOWED_SCOPES_BY_AUDIENCE
2. See SCOPE_AUDIENCE_MAPPING.md for valid scopes per audience
3. Verify user token includes required scope
4. User may need to re-authenticate with broader scope grant

**Remediation for insufficient_scope**:
1. User: Log in with broader scope grant
2. Or: Request narrower scope that user has
3. Check MCP server requirements for scope

### Server Errors

| Error Code | HTTP | OAuth Error | Description | Occurs When |
|------------|------|-------------|-------------|-------------|
| server_error | 500 | server_error | Unexpected internal error | Unhandled exception in exchange logic |
| temporarily_unavailable | 503 | temporarily_unavailable | Server unable to process request | PingOne or external service unavailable |

**Remediation for server_error**:
1. Check server logs for stack trace
2. Look for detailed error message with line number
3. If configuration-related, see config error remediations
4. If PingOne related, check PingOne status page
5. File bug report with error message and logs

**Remediation for temporarily_unavailable**:
1. Wait 30-60 seconds
2. Retry the operation
3. If persists, check:
   - Internet connectivity
   - PingOne status (status.pingone.com)
   - Firewall/proxy allowing PingOne access

## Error Response Format

All error responses follow RFC 8693 §5.2 format:

```json
{
  "error": "invalid_scope",
  "error_description": "The requested scope is invalid...",
  "error_code": "invalid_scope",
  "error_uri": "https://docs.example.com/errors/invalid_scope"
}
```

**Fields**:
- **error**: RFC 8693 error code (required)
- **error_description**: Human-readable description (optional)
- **error_code**: Internal diagnostic code (custom field)
- **error_uri**: Link to detailed documentation (optional)

## Examples

### Example 1: Invalid Client

**Request**:
```bash
POST /api/exchange HTTP/1.1
Content-Type: application/x-www-form-urlencoded

grant_type=urn:ietf:params:oauth:grant-type:token-exchange
&subject_token=eyJhbGc...&client_id=wrong-id&client_secret=wrong-secret
```

**Response**:
```json
HTTP/1.1 401 Unauthorized

{
  "error": "invalid_client",
  "error_description": "Client authentication failed (e.g., unknown client...)",
  "error_code": "invalid_client"
}
```

**Diagnosis**: Client ID or secret incorrect. Check PingOne app configuration.

### Example 2: Invalid Scope

**Request**:
```bash
POST /api/exchange HTTP/1.1

grant_type=urn:ietf:params:oauth:grant-type:token-exchange
&subject_token=eyJhbGc...&audience=https://agent-gateway.com&scope=transfer:execute
```

**Response**:
```json
HTTP/1.1 400 Bad Request

{
  "error": "invalid_scope",
  "error_description": "Requested scope invalid for this audience: transfer:execute not in [banking:agent:invoke, ai_agent]",
  "error_code": "invalid_scope"
}
```

**Diagnosis**: transfer:execute not valid for agent-gateway audience. See SCOPE_AUDIENCE_MAPPING.md.

## Quick Lookup

**I'm getting a 401**: Error is likely \`invalid_client\` or \`invalid_token\`. See Authentication Errors section.

**I'm getting a 403**: Error is likely \`unauthorized_client\` or \`access_denied\`. See Authorization Errors section.

**I'm getting a 400**: Error is likely \`invalid_scope\` or \`invalid_request\`. See Scope Errors section.

**I'm getting a 500**: Error is \`server_error\` or config-related. Check server logs and configuration.

## References

- RFC 8693 §5.2: https://tools.ietf.org/html/rfc8693#section-5.2
- SCOPE_AUDIENCE_MAPPING.md — Scope-audience validation
- CONFIGURATION_GUIDE.md — Configuration troubleshooting

## Threat Model

### Trust Boundaries

| Boundary | Protection |
|----------|-----------|
| Error Response → Operator | Structured error code for diagnostics |
| Error Message → Attacker | Description doesn't leak sensitive data |
| Error Log → Audit Trail | Event logged with category and context |

### STRIDE Register

| Threat | Type | Mitigation |
|--------|------|-----------|
| Error message injection | Tampering | Error codes standardized, descriptions predefined |
| Information disclosure via error | Info | Error messages generic, no secrets in logs |
| Denial via error spam | Denial | Error logging tracks frequency, alerts on spikes |
