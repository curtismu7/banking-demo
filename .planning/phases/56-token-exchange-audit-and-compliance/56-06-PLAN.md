## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test Input → Mocked OAuth | Tests inject OAuth service responses (mocked tokens, error responses) |
| Configuration → Exchange Logic | Configuration validation output consumed by _performTwoExchangeDelegation |
| Token Events → Test Assertions | Tests verify event stream matches expected RFC 8693 flow |

### STRIDE Register

| ID | Category | Component | Disposition | Mitigation |
|----|-----------|-----------|-----------|----|
| T-56-06-01 | I (Information Disclosure) | Test fixtures contain test credentials | Accept | Test credentials are non-sensitive (test-audience-01, etc); no real tokens in test files |
| T-56-06-02 | T (Tampering) | Mock OAuth could return invalid tokens | Mitigate | Tests verify token structure before use; claims are decoded and validated |
| T-56-06-03 | R (Repudiation) | Failed tests could be ignored | Mitigate | Test failure gates CI/CD pipeline (jest exit code 1); all test runs logged |
| T-56-06-04 | D (Denial of Service) | Malformed test data could crash parser | Mitigate | Tests use controlled JWT fixtures; error cases explicitly tested as "invalid_grant" responses |
