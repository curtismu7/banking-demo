# Phase 64-01: Unified Configuration Page Implementation

## Goal

Create a single, comprehensive configuration page that consolidates all demo settings from the current `/demo-data` and `/config` pages, providing users with a unified interface for managing the entire application configuration.

## Success Criteria

1. **Single Page Experience**: All configuration settings accessible from one unified page with logical section organization
2. **Complete Coverage**: 100% of configStore keys have corresponding UI controls with proper validation
3. **Seamless Migration**: Existing configurations preserved during transition with no data loss
4. **Improved UX**: Clear section organization, consistent validation, and comprehensive error handling
5. **Documentation Alignment**: Updated documentation reflecting the new unified configuration structure

## Tasks

### UNIFIED-01: Configuration Audit and Analysis

**Objective**: Complete comprehensive audit of all configuration keys and UI coverage

**Implementation Details**:
- **ConfigStore Analysis**: Extract complete list of all keys from `configStore.js` with setters/getters
- **UI Inventory**: Catalog all controls in `ConfigPage.js` and `DemoDataPage.js`
- **Gap Analysis**: Identify (a) keys with no UI, (b) UI controls with no backend, (c) orphaned settings
- **Environment Variable Mapping**: Verify env-var-backed keys use `configStore.getEffective()` at runtime
- **Dependency Mapping**: Document relationships between configuration settings

**Key Files**:
- `banking_api_server/services/configStore.js` - Full key registry analysis
- `banking_api_server/routes/config.js` - Patchable keys via API
- `banking_api_server/routes/demoData.js` - Demo-data API endpoints
- `banking_api_ui/src/components/ConfigPage.js` - Current UI controls inventory
- `banking_api_ui/src/components/DemoDataPage.js` - Current UI controls inventory

**Verification**:
- Complete matrix of all configStore keys with UI coverage status
- Identified gaps and orphaned settings documented
- Runtime usage verification for critical configuration paths

### UNIFIED-02: Backend API Consolidation

**Objective**: Create unified configuration API endpoints

**Implementation Details**:
- **Single Endpoint**: Design `GET/PUT/PATCH /api/admin/config` as unified configuration API
- **Schema Definition**: Create comprehensive JSON schema for all configuration keys
- **Validation Logic**: Implement consistent validation across all configuration types
- **Permission Model**: Ensure proper access control for sensitive settings
- **Migration Path**: Maintain backward compatibility during transition

**API Design**:
```javascript
// GET /api/admin/config - Returns full configuration schema
{
  "pingoneEnvironment": { ... },
  "tokenExchange": { ... },
  "agentMcp": { ... },
  "featureFlags": { ... },
  "demoScenario": { ... },
  "developerDebug": { ... }
}

// PATCH /api/admin/config - Partial updates
{
  "tokenExchange": {
    "mcpResourceUri": "https://mcp-server.pingdemo.com",
    "authMethods": { ... }
  }
}
```

**Key Files**:
- `banking_api_server/routes/unifiedConfig.js` - New unified configuration route
- `banking_api_server/middleware/configValidation.js` - Configuration validation middleware
- `banking_api_server/schemas/configSchema.js` - JSON schema definitions

**Verification**:
- All configuration keys accessible via unified API
- Proper validation and error handling
- Backward compatibility maintained
- Performance testing for large configuration payloads

### UNIFIED-03: Frontend Unified Configuration Page

**Objective**: Create comprehensive unified configuration UI

**Implementation Details**:
- **Component Architecture**: Design modular component structure for different configuration sections
- **Section Organization**: Implement logical grouping with collapsible sections
- **Form Management**: Use React Hook Form or similar for consistent form handling
- **Real-time Validation**: Implement client-side validation with server-side confirmation
- **State Management**: Proper loading states, error handling, and success feedback

**Section Structure**:

**Key Components**:

**Key Files**:

- `banking_api_ui/src/hooks/useUnifiedConfig.js` - Configuration data hook
- `banking_api_ui/src/styles/UnifiedConfig.css` - Styling for unified page

**Verification**:
- All configuration sections render properly
- Form validation works correctly
- Save/load functionality preserves data
- Responsive design works on different screen sizes
