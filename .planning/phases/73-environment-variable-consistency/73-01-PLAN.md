# Phase 73-01: Environment Variable Consistency — Plan

## Goal

Single authoritative `.env.example` that works for both localhost and Vercel. All URLs derived from `PUBLIC_APP_URL`. Startup validation catches missing vars early.

## Success Criteria

1. Fresh-clone developer can `cp .env.example .env`, fill secrets, and `npm start` with no 500 errors
2. All redirect URIs derived from `PUBLIC_APP_URL` — zero hardcoded hostnames
3. `check-env.js` startup validator lists all missing required vars clearly
4. Vercel production env vars match `.env.example` key names exactly
5. `.env` local file uses `PUBLIC_APP_URL=http://localhost:3001` pattern

---

## Tasks

### ENV-01: Audit all env var references

**Objective**: Inventory every `process.env.*` reference in `banking_api_server/`

**Tasks**:
- [ ] Run grep to collect all `process.env.` references
- [ ] Classify each as: required / optional / build-time
- [ ] Identify vars present in Vercel but missing from `.env.example`
- [ ] Identify vars in `.env` with hardcoded localhost values that should use `PUBLIC_APP_URL`

**Verification**:
- Complete list produced with classification

### ENV-02: Update `.env` and `.env.example`

**Objective**: Make `.env.example` the canonical, complete, documented template

**Tasks**:
- [ ] Replace all hardcoded `api.pingdemo.com:3002` redirect URIs with `${PUBLIC_APP_URL}/api/auth/oauth/...` pattern
- [ ] Add all missing vars identified in ENV-01 to `.env.example` with comments
- [ ] Group vars by category with section headers
- [ ] Mark required vs optional clearly
- [ ] Update `.env` local file to use `PUBLIC_APP_URL=http://localhost:3001` (not port 3002)
- [ ] Ensure `PINGONE_ADMIN_REDIRECT_URI` and `PINGONE_USER_REDIRECT_URI` use `PUBLIC_APP_URL`

**Example `.env.example` pattern**:
```bash
# ── Public App URL ─────────────────────────────────────────────
# Sets the base URL for all OAuth redirect URIs.
# localhost:  PUBLIC_APP_URL=http://localhost:3001
# Vercel:     PUBLIC_APP_URL=https://banking-demo-puce.vercel.app
PUBLIC_APP_URL=http://localhost:3001

# Derived — these are auto-constructed by the server from PUBLIC_APP_URL
# but you can override them explicitly here if needed:
# PINGONE_ADMIN_REDIRECT_URI=${PUBLIC_APP_URL}/api/auth/oauth/callback
# PINGONE_USER_REDIRECT_URI=${PUBLIC_APP_URL}/api/auth/oauth/user/callback
```

**Verification**:
- `diff .env .env.example` shows only secret values differ, not structure

### ENV-03: Create `scripts/check-env.js` startup validator

**Objective**: Fail fast with a clear error when required vars are missing

**Tasks**:
- [ ] Create `banking_api_server/scripts/check-env.js`
- [ ] List all `REQUIRED_VARS` by category
- [ ] Warn (not fail) on `OPTIONAL_VARS` that have no default
- [ ] Print a formatted table of missing vars with description
- [ ] Exit non-zero on any missing required var in production
- [ ] Skip in test environment (`NODE_ENV=test`)

**Example output**:
```
❌ Missing required environment variables:
  PINGONE_ENVIRONMENT_ID   — Your PingOne tenant environment UUID
  SESSION_SECRET           — Express session signing secret (min 32 chars)

Run: cp .env.example .env and fill in the values above.
```

**Verification**:
- `node scripts/check-env.js` without `.env` exits 1 with clear output
- With complete `.env` exits 0
