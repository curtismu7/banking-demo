  <action>
Create a new `run.sh` script at repo root that:

1. **Subcommands:**
   - `./run.sh start` — Start all services (UI, API, MCP server)
   - `./run.sh stop` — Stop all services gracefully
   - `./run.sh restart` — Stop then start
   - `./run.sh logs` — Tail logs from all services
   - `./run.sh test` — Run full test suite
   - `./run.sh status` — Show status of all services (running/stopped, PIDs)
   - `./run.sh help` — Show usage

2. **Pre-flight checks (start subcommand):**
   - Check Node.js is installed and version >= 16
   - Check npm is installed
   - Check required env vars exist (.env files)
   - Check ports are available (3000, 3001, 4000, 3002, etc.)
   - Warn if node_modules missing and suggest npm install

3. **Process Management:**
   - Create `.pids/` directory for PID files
   - Store PIDs: `.pids/ui.pid`, `.pids/api.pid`, `.pids/mcp.pid`
   - Use PIDs to track running processes
   - Graceful shutdown: send SIGTERM, wait 5s, then SIGKILL if needed

4. **Post-start banner:**
   - Show all services started with PIDs
   - Show URLs (http://localhost:3000, http://localhost:3001/api, etc.)
   - Show log file locations
   - Show how to view logs: `./run.sh logs`

5. **Code quality:**
   - Pass shellcheck without warnings
   - Use set -e for error handling
   - Quote all variables
   - Comment all major sections
   - Handle signals (SIGTERM, SIGINT) for clean shutdown

6. **Integration:**
   - Preserve all functionality from current scripts (start, stop, tests, config updates, upstash)
   - In start subcommand: call banking_api_server/setup-env.sh if needed
   - In test subcommand: run npm test in all three services
   - Optional: upstash reset/update as subcommands or separate utility

Output: Executable `/run.sh` at repo root
