<objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env vars → configStore.getEffective() | Process env vars are set by deployment platform (Vercel, local .env) — trusted but must not leak secrets to client |
| configStore → services | Internal service-to-service — trusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-88-01 | Information Disclosure | .env new SECRET entries | mitigate | PINGONE_WORKER_CLIENT_SECRET, PINGONE_ADMIN_APP_CLIENT_SECRET, PINGONE_USER_APP_CLIENT_SECRET added to SENSITIVE_KEYS array in configStore.js so they are never exposed via /api/config public endpoint |
| T-88-02 | Spoofing | bffClientId in may_act injection | mitigate | Fix replaces process.env worker token (wrong identity) with correct admin_client_id — prevents wrong client_id being injected into may_act claim |
| T-88-03 | Elevation of Privilege | 2-exchange errors without httpStatus | accept | This is a test visibility bug, not an auth bypass; correct 503/502 status codes ensure callers fail-closed when credentials are missing |
</threat_model>
