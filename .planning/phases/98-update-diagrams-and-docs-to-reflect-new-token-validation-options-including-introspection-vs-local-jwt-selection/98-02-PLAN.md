Create `docs/token-validation-flow.drawio` as a draw.io XML diagram showing the token validation decision flow. The diagram MUST be draw.io XML format (not Mermaid, not ASCII).

The diagram should show:

**Left side — Operator Config Path:**
- Demo Operator → Config Page (Token Validation Tab) → POST /api/config/validation-mode → validationModeConfig.js (in-memory store)
- Label: "Runtime mode switch — no restart needed"

**Center — API Request Path:**
- API Request arrives at BFF (banking_api_server)
- BFF reads validationModeConfig.getValidationMode()
- Decision diamond: "Mode = introspection?"

**Right branch (YES — Introspection):**
- POST /as/introspect (RFC 7662) to PingOne
- PingOne returns { active: true/false, sub, scope, aud }
- Cache result 30s (INTROSPECTION_CACHE_TTL)
- Pros label: "✓ Detects revoked tokens  ✓ Real-time  ✗ Network call"

**Left branch (NO — JWT local):**
- Fetch JWKS (cached) from PingOne /as/jwks
- Verify RS256 signature locally (RFC 7519)
- Check exp, aud, iss claims
- Pros label: "✓ ~1ms  ✓ Offline capable  ✗ Won't detect revoked"

**Bottom — Both paths converge:**
- Extract scopes from validated token
- requireScopes middleware → allow/deny

**Health check side-box:**
- GET /api/health/introspection box
- Operator tests PingOne connectivity before selecting introspection mode
- Shows: 200 connected | 503 failed
