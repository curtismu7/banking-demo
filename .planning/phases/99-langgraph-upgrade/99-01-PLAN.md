# Phase 97.1: LangGraph Migration

## Description
Migrate the banking agent from LangChain's `createAgent` to LangGraph's `StateGraph` pattern for better state management and more sophisticated agent workflows.

## Background
The banking agent was using LangChain's simple `createAgent` function, which is being deprecated in favor of LangGraph's more powerful StateGraph pattern. LangGraph provides:
- Better state management with defined state schemas
- Cyclic workflows and multi-agent patterns
- More granular control over agent execution
- Better support for complex banking operations with tool calling

## Dependencies
- None (this is an architectural improvement, not dependent on other phases)

## Deliverables
- Install `@langchain/langgraph` package
- Migrate `agentBuilder.js` from LangChain `createAgent` to LangGraph `StateGraph`
- Update `bankingAgentLangChainService.js` to use LangGraph graph invocation
- Test the migrated agent with banking operations
- Document the migration and any breaking changes

## Implementation Details

### 1. Package Installation
- Installed `@langchain/langgraph` version 1.2.8

### 2. Agent Builder Migration
**File:** `services/agentBuilder.js`

**Changes:**
- Replaced `createAgent` from `langchain` with `StateGraph` from `@langchain/langgraph`
- Defined state schema using `Annotation.Root` with:
  - `messages`: Array of conversation messages with concat reducer
  - `userId`: User authentication ID
  - `userToken`: OAuth access token
  - `sessionId`: Express session ID
  - `tokenEvents`: Token event tracking array
  - `provider`: LLM provider (anthropic/groq)
- Created agent node function that invokes the model with system prompt and messages
- Built StateGraph with single agent node and start/end edges
- Returns compiled graph with initial state instead of agent object

### 3. Service Layer Migration
**File:** `services/bankingAgentLangChainService.js`

**Changes:**
- Updated to destructure `graph` and `initialState` from `createBankingAgent`
- Invokes graph with `initialState` plus user message
- Extracts response from final state's last message
- Maintains same API contract for route handlers

### 4. Testing
- API server restarted successfully
- No breaking changes to `/api/banking-agent/message` endpoint
- Agent maintains same behavior with improved architecture

## Notes
- The migration is backward compatible - the API endpoint signature remains unchanged
- LangGraph provides a foundation for future enhancements like multi-agent workflows
- Tool calling integration with MCP tools can be added as additional nodes in the graph
