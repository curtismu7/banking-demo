  </read_first>
  <action>
    Start the local development servers. The standard way (ports 3000/3001):

    ```
    # Terminal 1 — API server
    cd banking_api_server && npm start

    # Terminal 2 — UI dev server
    cd banking_api_ui && npm start
    ```

    OR use run-bank.sh (ports 3002/4000):
    ```
    ./run-bank.sh
    ```

    Watch startup output for:
    - ✅ "Banking API server running on port 3001" (or 3002)
    - ✅ No "Error:" or "FATAL" lines
    - ✅ configStore loads without errors
    - ✅ No "Cannot find module" or missing env var crashes

    Common issues:
    - `KV_REST_API_URL` not set → Upstash store init fails (expected locally; in-memory fallback is OK)
    - Missing `.env` in banking_api_server → copy from `env.example` and fill required PingOne vars
    - Port already in use → kill prior process: `lsof -ti:3001 | xargs kill -9`

    If the server crashes on startup, read the stack trace and fix the root cause before the next task.
  </action>
