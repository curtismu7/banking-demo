# GSD New Workflow

## Purpose
Create new milestones, phases, or workstreams for structured development planning.

## When to Use
- Starting a new development milestone
- Creating a new phase in existing milestone
- Setting up parallel workstreams
- Beginning structured development work

## Steps

### 1. Determine Creation Type
// turbo
Identify what type of structure to create:
```bash
echo "What to create: milestone, phase, or workstream?"
```

### 2. Create Milestone
For new milestones:
- **Name**: Choose descriptive milestone name
- **Description**: Define milestone objectives
- **Phases**: Plan initial phases
- **Structure**: Create milestone directory structure

### 3. Create Phase
For new phases within existing milestone:
- **Identify**: Select parent milestone
- **Define**: Phase objectives and scope
- **Plan**: Break down into tasks
- **Integrate**: Ensure phase fits milestone

### 4. Create Workstream
For parallel development:
- **Purpose**: Define workstream focus
- **Scope**: Set workstream boundaries
- **Integration**: Plan coordination points
- **Timeline**: Define workstream duration

### 5. Setup Structure
- **Directories**: Create required directory structure
- **Planning**: Initialize planning documents
- **Configuration**: Set up GSD configuration
- **Documentation**: Create initial documentation

## Expected Outcome
- Clear development structure
- Proper planning setup
- Defined objectives and scope
- Ready for development work

## Notes
- Use clear, descriptive names
- Define measurable objectives
- Plan integration points early
- Update planning documents promptly
