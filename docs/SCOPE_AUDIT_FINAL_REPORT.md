## Executive Summary

A comprehensive audit of OAuth scope configuration reveals a **critical issue**: the floating agent (and agent-triggered actions) fail with "Could not parse: undefined" because:

1. **PingOne user tokens lack banking scopes** — New OAuth login grants only generic scopes (`openid`, `profile`, `email`, `offline_access`)
2. **Code expects banking-specific scopes** — RFC 8693 integration requires `banking:read`, `banking:write`, or similar for MCP token exchange
3. **Scope validation fails silently** — When token exchange insufficient scopes, error thrown without proper `.message` property
4. **Resource-audience mappings misconfigured** — ALLOWED_SCOPES_BY_AUDIENCE uses example.com URIs; actual PingOne resources have different URIs

**Action Required:** Add banking scopes to your PingOne OAuth user application(s), or configure the user app to request scoped access to a PingOne Resource Server.

## Root Cause Analysis

### Floating Agent Error: "Could not parse: undefined"

**Error Flow:**
```
User logs in (new session, incognito) 
  ↓
User token generated by PingOne with scopes: [openid, profile, email, offline_access]
  ↓
User attempts agent action: "Show my accounts"
  ↓
POST /api/banking-agent/message
  ↓
Token exchange triggered (RFC 8693)
  ↓
agentMcpTokenService validates: MIN_USER_SCOPES_FOR_MCP_EXCHANGE (default: 1)
  ↓
Check: User token has non-OIDC scopes? (banking:read, banking:write, etc.) → **NO**
  ↓
Error thrown without `.message` property
  ↓
Backend catches error → res.status(500).json({ error: undefined })
  ↓
Frontend displays: "Could not parse: undefined"
```

**Severity:** 🔴 **BLOCKER** — Agent is non-functional on fresh login

## Finding 3: Code vs. PingOne Mismatch

### **ALLOWED_SCOPES_BY_AUDIENCE Configuration**

The code defines hardcoded scope mappings in `configStore.js`:

```javascript
const ALLOWED_SCOPES_BY_AUDIENCE = {
  'https://agent-gateway.example.com': ['banking:agent:invoke', 'ai_agent'],
  'https://ai-agent-gateway.example.com': ['banking:read', 'banking:write', 'banking:agent:invoke'],
  'https://mcp-gateway.example.com': ['banking:mcp:invoke', 'mcp_resource_access'],
  'https://resource.example.com/mcp': ['get_accounts:read', 'transfer:execute', 'check:read'],
};
```

**Problem:** These are **example.com URIs**, not your actual PingOne resource URIs.

**Required Fix:** The code must either:
- **Option A:** Use actual resource URIs from PingOne config, OR
- **Option B:** Read ALLOWED_SCOPES_BY_AUDIENCE from environment/configStore at startup

## Remediation Plan

### **Issue 1: User Tokens Lack Banking Scopes** 🔴 BLOCKER

**Root Cause:** PingOne user OAuth app not configured to grant banking-specific scopes.

**Solutions:**

#### **Option A: Add Scopes to OIDC App (Simple)**
1. In PingOne admin console, go to **Integrations > Applications** → Your user OAuth app
2. Go to **OAuth Resource** tab
3. Verify **"PingOne API"** is listed
4. Find **"Super Banking Banking API"** resource (or create it if missing)
5. Click **"Add Scopes"** and add:
   - ✅ `banking:read` (required)
   - ✅ `banking:write` (required for transfers/deposits)
   - ✅ `ai_agent` (required for agent)
   - Optional: `banking:accounts:read`, `banking:transactions:read`, `banking:transactions:write`
6. Save
7. **Test:** Log out, log in fresh, check user token scopes in `/api/session/preview`

#### **Option B: Create a Custom Resource Server (Recommended)**
1. Create a PingOne Resource Server named **"Super Banking Banking API"**
2. Set resource URI: `https://banking-api.banking-demo.com` (or your domain)
3. Add resource scopes: `banking:read`, `banking:write`, `ai_agent`, and others
4. Configure user OAuth app to request access to this resource
5. Update code: Set `ENDUSER_AUDIENCE=https://banking-api.banking-demo.com` in `.env`
6. Update `configStore.js` ALLOWED_SCOPES_BY_AUDIENCE to use actual URIs

**Estimated Effort:** Option A = 5 min; Option B = 15 min

**Test Criteria:**
```bash
# After fix, log in fresh and check:
curl http://localhost:3001/api/session/preview | jq '.userToken.scopes'
# Should include: ["openid", "profile", "email", "offline_access", "banking:read", "banking:write", "ai_agent"]
```
