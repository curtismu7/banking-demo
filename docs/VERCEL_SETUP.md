# Vercel Deployment Setup Guide

Complete guide for deploying the BX Finance Banking Demo to Vercel.

---

## Quick start

```bash
# Interactive wizard — collects, validates, and pushes all env vars
node scripts/setup-vercel-env.js

# Validate existing config (no changes)
node scripts/setup-vercel-env.js --check

# Then deploy
vercel --prod
```

---

## Required environment variables

### 1. Session store (Upstash Redis REST)

OAuth requires shared sessions across Vercel serverless instances. Get free credentials at [upstash.com](https://upstash.com).

```
UPSTASH_REDIS_REST_URL=https://<id>.upstash.io
UPSTASH_REDIS_REST_TOKEN=<token>
# KV_REST_API_* are aliases for the same credentials:
KV_REST_API_URL=https://<id>.upstash.io
KV_REST_API_TOKEN=<token>
SESSION_SECRET=<64-char hex>   # generated by the wizard
```

Alternatively: Vercel Dashboard → Storage → Connect KV Store (auto-injects credentials).

### 2. PingOne OAuth

```
PINGONE_ENVIRONMENT_ID=<env-id>
PINGONE_REGION=com                        # or eu / ca / asia
PINGONE_AI_CORE_CLIENT_ID=<admin-client-id>
PINGONE_AI_CORE_CLIENT_SECRET=<secret>
PINGONE_AI_CORE_REDIRECT_URI=https://<your-domain>/api/auth/oauth/callback
PINGONE_AI_CORE_USER_CLIENT_ID=<user-client-id>
PINGONE_AI_CORE_USER_CLIENT_SECRET=<secret>
PINGONE_AI_CORE_USER_REDIRECT_URI=https://<your-domain>/api/auth/oauth/user/callback
REACT_APP_CLIENT_URL=https://<your-domain>
PUBLIC_APP_URL=https://<your-domain>
FRONTEND_ADMIN_URL=https://<your-domain>/admin
FRONTEND_DASHBOARD_URL=https://<your-domain>/dashboard
```

### 3. MCP server URL

```
MCP_SERVER_URL=wss://your-mcp-server.example.com
```

The MCP server cannot run on Vercel (requires persistent WebSocket). Deploy `banking_mcp_server/` to Railway, Render, or Fly.io.

## MCP token security — MCP_RESOURCE_URI

**This is the most commonly missed configuration step.**

When the BFF performs an RFC 8693 token exchange to obtain a token for the MCP server, it sends the `audience` parameter. The MCP server validates that incoming tokens carry that audience in their `aud` claim. Both sides must agree on the same value.

| Variable | Where it lives | Purpose |
|---|---|---|
| `MCP_RESOURCE_URI` | `banking_api_server` (Vercel) | Sent as `audience` in RFC 8693 token exchange |
| `MCP_SERVER_RESOURCE_URI` | `banking_mcp_server/.env` | Validates `aud` claim on inbound tokens |

Both must be set to the **same HTTPS URL** — the public URL of your MCP server.

### Setting MCP_RESOURCE_URI on Vercel

**Via the setup wizard (recommended):**
```bash
node scripts/setup-vercel-env.js
# Step 5 will prompt for the value and push both vars
```

**Via Vercel CLI directly:**
```bash
MCP_URI="https://your-mcp-server.example.com"

echo "$MCP_URI" | vercel env add MCP_RESOURCE_URI production --yes --force
echo "$MCP_URI" | vercel env add MCP_SERVER_RESOURCE_URI production --yes --force
```

**Via Vercel Dashboard:**
1. Project → Settings → Environment Variables
2. Add `MCP_RESOURCE_URI` = `https://your-mcp-server.example.com`
3. Add `MCP_SERVER_RESOURCE_URI` = same value
4. Redeploy

### Setting MCP_SERVER_RESOURCE_URI on the MCP server

In `banking_mcp_server/.env`:
```
MCP_SERVER_RESOURCE_URI=https://your-mcp-server.example.com
```

On Railway/Render/Fly: add it as an environment variable in their dashboard. Same value as above.

### What happens without this

| Scenario | Effect |
|---|---|
| Neither var set | Token exchange skipped; MCP server accepts any token (no `aud` validation) |
| Only BFF side set | BFF sends `audience` in exchange; MCP server skips `aud` validation |
| Only MCP server side set | BFF sends no audience; MCP server rejects tokens with `aud` mismatch |
| **Both set (correct)** | **BFF requests token with audience; MCP server validates it — full RFC 8707 enforcement** |

## RFC 8693 token exchange (on_behalf_of)

When a user invokes the banking agent, the BFF exchanges the user's session token for an MCP-scoped token using RFC 8693. This requires an additional OAuth client.

```
AGENT_OAUTH_CLIENT_ID=<client-id>          # OAuth client authorized for token exchange
AGENT_OAUTH_CLIENT_SECRET=<secret>
BFF_CLIENT_ID=<admin-client-id>            # Often same as PINGONE_AI_CORE_CLIENT_ID
USE_AGENT_ACTOR_FOR_MCP=true               # Adds act claim identifying the agent
REQUIRE_MAY_ACT=false                      # Whether to enforce may_act claim
```

## Post-deploy checklist

1. **PingOne redirect URIs** — add to your PingOne app config:
   - Admin: `https://<your-domain>/api/auth/oauth/callback`
   - Customer: `https://<your-domain>/api/auth/oauth/user/callback`

2. **Redeploy** after all env vars are set:
   ```bash
   vercel --prod
   ```

3. **MCP server** — ensure `MCP_SERVER_RESOURCE_URI` is set in `banking_mcp_server/.env` (see above).

4. **Verify session store:**
   ```
   GET /api/auth/debug
   ```
   Expect:
   ```json
   {
     "sessionStoreType": "upstash-rest",
     "sessionStoreHealthy": true,
     "sessionRestored": false
   }
   ```

5. **Sign out and sign in** — existing sessions (cookie-only) won't hydrate from Redis.

## Troubleshooting

| Symptom | Likely cause |
|---|---|
| `invalid_state` on OAuth callback | Session lost between instances — `UPSTASH_REDIS_REST_*` not set or wrong |
| "connecting…" on Agent panel forever | `MCP_SERVER_URL` not set or MCP server not running |
| `403` / `aud` mismatch in MCP server logs | `MCP_RESOURCE_URI` on BFF ≠ `MCP_SERVER_RESOURCE_URI` on MCP server |
| Token exchange returns empty audience | `MCP_RESOURCE_URI` not set on BFF (Vercel) |
| `sessionStoreType: "memory"` in `/api/auth/debug` | Redis credentials not loaded — redeploy after setting vars |

See also: `docs/runbooks/session-regression.md`, `banking_api_server/OAUTH_SCOPE_CONFIGURATION.md`
